Topic C

Programmability Concepts

Automation of network management is impossible without the ability to craft software code that performs repetitive tasks.

Web Services

Web services are applications or data sources that are accessible through the HTTP and HTTPS protocols and that provide interoperability, communication, and data exchange between applications. Messages between web services and other applications are most often created in Extensible Markup Language (XML), and the web services do not depend on a single operating system or programming language.

For example, consider a web page for a travel agency that provides travelers with the ability to make airline and hotel reservations, as well as reserve a rental car and buy travel insurance, all from the agency's web page. Each of these travel services is enabled on the web page by the use of web services. The agency can provide all this functionality without the need for writing application code.

Web services are often categorized according to the protocols and standards they use:

  • XML-RPC is the simplest XML-based protocol for exchanging information between networked computers. RPC stands for Remote Procedure Call.
  • Universal Description, Discovery, and Integration (UDDI) is an XML-based standard used for describing, publishing, and discovering web services. You can think of it as providing a catalog of available web services.
  • Service Oriented Architecture Protocol (SOAP) is an XML-based protocol that enables data exchange over HTTP or SMTP. It is steadily being deprecated in favor of REST.
  • Representational State Transfer (REST) is an architectural style for creating web services.

Representational State Transfer (REST) Technology

Representational State Transfer (REST) is a software architecture that defines constraints that are used during the creation of web services. These constraints provide performance, scalability, reliability, simplicity, modifiability, visibility, and portability.

Constraint Description
Client-server architecture This constraint separates the user interface from the data source.
Statelessness This constraint follows the client-server constraint, as the server does not retain client information after a request has completed. All the information required for a client request is sent within the request.
Cacheability This constraint requires responses to define themselves as cacheable or not cacheable to prevent clients from receiving incorrect or outdated data in response to a request.
Layered systems This constraint requires transparency for clients that are not directly connected to the end server. No additional code is required to address any intermediary servers that pass the client request to the end server. In addition, this constraint enables the addition of a security layer, protecting client/server communications without affecting the performance of the web services themselves.
Uniform interface Constraints to provide a uniform interface involve including resource identifiers for each individual resource being requested, self-description messages that provide the information needed to process the message, the ability for clients to manipulate resources, and the use of hyperlinks to discover available actions.
Code on demand (optional) This optional constraint enables servers to transfer code to a client to provide any additional or different functionality required to complete a request.

RESTful APIs

RESTful APIs are APIs that are created according to REST constraints and that use HTTP requests to manipulate data. They provide flexibility for developers because:

  • They use protocols that are already in place on an IP network, so there is no need to load additional libraries or software.
  • They decouple data from the underlying resources or methods so one RESTful API can meet the needs of many types of program calls.
  • They can use many data formats in addition to XML, including JavaScript Object Notation (JSON), and YAML Ain't Markup Language (YAML).
  • They use less bandwidth than SOAP technology.

HTTP Verbs

RESTful APIs leverage at least the following HTTP verbs:

  • POST to create a resource.
  • GET to read a resource.
  • PUT to update or create a resource.
  • DELETE to remove a resource.

These verbs correspond to the basic Create Read Update Delete (CRUD) operations that are commonly accepted user actions for data.

Data Encoding

In general computing, data encoding is the process of converting data from one form to another. When it comes to RESTful APIs, a common form of encoding is URL encoding. This is the practice of ensuring that the URLs that are sent over an IP network conform to the ASCII character set. For instance, there is no ASCII character for a space or for various punctuation marks that are often included in URLs, so encoding replaces those entities with the percent symbol, %, followed by two hex digits, such as %20 for a space.

Software Configuration Management Options

Software configuration management (SCM) enables developers and other IT personnel to organize the various facets of the software used by an organization, such as source code, documentation, problems, change requests, and changes that have been implemented. Common goals of SCM include configuration, identification, baselining, control, and change management.

There are numerous options for implementing configuration management for software. Some of the most notable include Puppet, Chef, and Ansible.

Puppet, Chef, and Ansible

This table describes three popular SCM tools.

Tool Description
Puppet® Puppet is an open-source SCM tool used primarily for deploying, configuring, and managing servers. Often used in cloud deployments, it employs a master/slave architecture to enable various node configurations to be transferred to the puppet master. Puppet uses a set of universal specifications to provide automation of administrative tasks such as adding users, installing software packages, and updating server configurations.
Chef® Chef is an automation tool for configuring and managing infrastructure as code. Chef is based on the Ruby programming language and provides recipes to record and update information about the infrastructure and any compliance rules that might be in effect.
Ansible® Ansible is a simple, open-source IT automation platform that is commonly used for websites and documentation. It uses a push-based architecture that sends configuration information in the form of Ansible modules to the nodes you want to manage and then runs the modules over SSH and removes them when the modules finish execution. Modules can be stored anywhere on your network - no servers or databases are needed.

JavaScript and JavaScript Object Notation (JSON)

JavaScript is a lightweight scripting language that is often used to provide dynamic content and interactivity on web pages and to develop mobile apps. It works with HTML and CSS and provides the automation found on most websites today.

JavaScript Object Notation (JSON) is an open standard text-based data format that uses human-readable text to transmit data objects and arrays in web applications. It is considered a low-overhead alternative to XML and is widely used to exchange data in web services. A JSON file contains a collection of name/value pairs and lists of values. (You will sometimes see references to key/value pairs or attribute/value pairs - these are just variations in name).

JSON is based on the same syntax as JavaScript, but it can be used with many other programming and scripting languages. Files have the extension .json and the MIME type application/json.

Another example sets up the data format for an online catalog.

JSON Encoded Data Interpretation

Because JSON files use human-readable text, you can open them using several types of programs:

  • You can import a JSON file into your web browser to view or modify it.
  • You can open a JSON file in Notepad or any other text editor to view or modify it.
  • You can import a JSON file into Excel, or into programs created in languages such as Python, Java, or JavaScript.
  • You can download a JSON viewer, editor, or parser and use that to open JSON files.
