Describe Network Topology Architectures

Topic C

Network Topologies

Once you connect network components together, you need to ensure they are able to communicate. By identifying rules for physical layout, how nodes are connected, signaling, and data flow, you can ensure the transfer of data from node to node.

Physical and Logical Topologies

Network topologies are specifications that determine the network's overall layout, signaling, and data-flow patterns. They are classified into:

  • Physical topology: A topology that describes a network's physical wiring layout or shape. Some of the physical topologies are bus, star, ring, mesh, and hybrid.
  • Logical topology: A topology that describes the paths through which data moves.

The physical and logical topologies for a given network can be, and often are, different.

Small Office/Home Office (SOHO) Networks

A small office/home office (SOHO) network is a group of computers (typically fewer than 10), along with peripherals such as printers and networking devices such as switches and APs, that are connected together by cables or wireless connections for the purpose of conducting business activities.

Each SOHO network is as individual as the business that they support. An author who works from home might have several laptops, a printer or multifunction device, voice over IP (VoIP) telephone equipment, and a SOHO (broadband) router, while a single-site engineering office might add plotters, specialized workstations for computer-aided design (CAD) projects, and a server to provide file storage and email services.

Tiered Networks

To facilitate network implementation and management, you can implement a tiered network approach. The key concept behind this approach is to group devices into two or more tiers, so that devices in the same tier all have the same role. Once the role of each device has been established, you can define the interconnections between devices. The most common implementation are two-tier and three-tier network architectures.

  • Two-tier architecture is similar to a basic client-server model. The application at the client end directly communicates with the data on the server end. The client runs the user interfaces and application programs, while the server stores the data.
  • In a three-tier architecture, the data access, data storage, and user interface functions are developed and maintained as independent modules on separate platforms.

Cisco uses a three-tier hierarchical model to build networks that are scalable, secure, and reliable. The tiers are called the Access layer, the Distribution layer, and the Core layer. When network services and functions are separated into functional layers, making a change or adding to any one function should not require a total rebuild of the network, just a modification of the affected part.

  • The Core layer is basically the network backbone. Consisting of high-speed switching devices and cables, this layer is responsible for fast and reliable data transmission.
  • The Distribution layer provides LAN-based routing, packet filtering, and packet manipulation, among other functions.
  • The Access layer is where clients connect to the network.

The three-tier architecture is ideal for larger enterprises that have multiple buildings on a single campus. Smaller organizations might find the number of connections required to provide a full mesh between the Core and the Distribution layers to be more effort than the perceived benefits of scalability. These organizations would likely find it more advantageous to implement a two-tier architecture.

Cisco Collapsed Core Architecture

The three-tier hierarchical approach can be modified to use two tiers, often referred to as the Cisco collapsed core architecture. In this scenario, the Core and Distribution layers are combined into one layer. This reduces network costs while maintaining most of the benefits of the three-tier architecture. It also results in the functions of the Distribution layer and Core layer being implemented in a single device.

Spine-Leaf Topology

One implementation of a collapsed core architecture uses a spine-and-leaf topology. The spine layer contains the devices in the Core layer (network backbone), while the leaf layer contains the lower-tier switches. Each leaf is connected to each device in the spine layer in a mesh arrangement, meaning that each switch has a direct connection to each of the spine-layer devices. This facilitates balancing the traffic to the backbone, and it provides redundancy in the case of failure of a Core-layer device.

With a spine-and-leaf architecture, no matter which leaf switch to which a server is connected, its traffic always has to cross the same number of devices to get to another server (unless the other server is located on the same leaf). This approach keeps latency at a predictable level because a payload only has to hop to a spine switch and another leaf switch to reach its destination.

WAN Topologies

Interconnecting multiple sites across WANs can involve different service provider technologies and WAN topologies. Common WAN topologies are:

  • Point-to-point topology
  • Hub-and-spoke topology
  • Full mesh topology
  • Partial mesh topology
  • Single-homed topology
  • Dual-homed topology
  • Multi-homed topology

These WAN topologies are described in the following table.

WAN Topology Description
Point-to-point The simplest of the WAN topologies, in which a customer can connect two sites by using either a dedicated telecommunications link or a VPN across the Internet.
Star (Hub-and-spoke)
  • A WAN topology where the central site acts as a hub for each remote (leaf) site.
  • Remote (leaf) sites can communicate with each other by routing through the central hub site, or they can be restricted to communicating only with the central site, not with other remote (leaf) sites.
  • The central site serves as the root of a tree, and each remote site is one of the leaves.
  • The topology is also referred to as hub-and-spoke and point-to-multipoint. Regardless of the term you use for the topology, it creates a service that works well for networks that have a central site along with many remote sites.
Full mesh
  • A WAN topology that provides each site with a direct connection to every other site.
  • This topology is fully redundant and the most expensive to implement.
  • This topology is useful if the enterprise needs to connect several sites to the WAN with a goal of allowing every site to send frames directly to every other site.
Partial mesh
  • A hub-and-spoke WAN topology in which some of the leaf sites also have links to each other, as well as to the central hub site.
  • This topology is useful if the enterprise wants to allow some leaf sites to directly communicate with each other, without the latency or potential congestion of passing traffic through a central hub site.
  • This topology is also referred to as a redundant star.
Single-homed
  • Provides a single link between the Internet Service Provider (ISP) and the enterprise network. There must be only one ISP, and this link must be the only connectivity to that ISP.
  • Typically used to connect an enterprise branch office to the Internet.
  • Enables various kinds of WAN links such as Digital Subscriber Line (DSL), cable, fiber Ethernet, or a wireless Long-Term Evolution (LTE) connection.
  • If the link to the ISP fails or the router at either end of the link fails, the Internet will not be accessible.
Dual-homed
  • Provides two links between the ISP and the enterprise network. There must be only one ISP, and each link must provide one of the two paths to reach the ISP.
  • Using two links provides an alternative method for reaching the ISP if either link fails.
Multi-homed
  • Provides links between two ISPs and the enterprise network.
  • Single multi-homed designs have one router at the edge of the enterprise network that is connected to two different ISPs, or it has two enterprise routers, each of which are connected to different ISPs.
  • Dual multi-homed designs provide redundant links to two different ISPs. If one enterprise router is used, it has two links to each ISP. If two enterprise routers are used, each can have two links to one of the ISPs, or for additional redundancy, you can configure the routers so each has a link to both ISPs.

WAN Technologies Supported by Cisco

Cisco IOS software supports a range of WAN capabilities that will fit the need of most of the network environments. Some of the wide area network (WAN) technologies supported by Cisco are:

  • Virtual private network (VPN): A set of security protocols that are implemented on both ends of an insecure network, such as the Internet, to provide secure connectivity to remote sites and partner sites.
  • Frame Relay: A standard Data Link protocol that defines the capabilities to create a frame-switched or packet-switched service, which allows Data Terminal Equipment (DTE) devices like routers to send data to many other devices using a single physical connection to the Frame Relay.
  • Asynchronous Transfer Mode (ATM): A cell-switching and multiplexing technology that provides data link (Layer 2) services with scalable bandwidth usually over Synchronous Optical Networking (SONET)/Synchronous Digital Hierarchy (SDH) physical (Layer 1) links.

On-Premises and Cloud Services

Another characteristic of network topologies defines the location of network services.

  • On-premises services are those network services that are housed and maintained at an organization's physical site.
  • Cloud services are network services housed and maintained remotely and accessed through WAN links. Public clouds are provided by a third party (cloud service provider) on a subscription basis, while private clouds are created and maintained by an organization for its own use. There is also a hybrid cloud approach that you can implement - for example, you could subscribe to a software service from a cloud service provider, but store sensitive data on a private cloud.

Cloud Service Providers

Some of the services and benefits offered by cloud service providers include:

  • On demand self service.
  • Broad network access.
  • Resource pooling (multi-tenancy).
  • Rapid elastic provisioning.
  • Measured services (pay-for-what-you-use).
