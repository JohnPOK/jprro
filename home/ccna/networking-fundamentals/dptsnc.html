Describe Protocols That Support Network Communications

Topic E

Basic Networking Protocols

Network communication is impossible without the proper use of networking protocols. There are scores of protocols addressing various types of communication, and you need to be familiar with those used on your network.

Protocols

A protocol is a standard set of rules that enable electronic devices to communicate. Protocols specify how information should be structured, as well as how the information should be sent and received by each device.

Various protocols are associated with each layer of the OSI or TCP/IP model, such as:

  • Media access and delivery protocols
  • Addressing and routing protocols
  • Transport protocols
  • Application protocols
  • Note: Some protocols are multilayer and do not easily fit into one of these four designations.

Media Access and Delivery Protocols

Media access and delivery protocols are those that operate at the Network Interface layer of the TCP/IP model (and the Physical and Data Link layers of the OSI model). These protocols and standards include:

  • Ethernet and its variations, such as Fast Ethernet and Gigabit Ethernet.
  • Asynchronous Transfer Mode (ATM).
  • Token Ring.
  • Fiber Distributed Data Interface (FDDI).
  • Point-to-Point Protocol (PPP)

Addressing and Routing Protocols

Addressing and routing protocols are those that operate at the Network layer of the TCP/IP model (and the Network layer of the OSI model). These protocols and standards include:

  • Internet Protocol (IP), including both IPv4 and IPv6.
  • Address Resolution Protocol (ARP).
  • Internet Control Message Protocol (ICMP).
  • Open Shortest Path First (OSPF).
  • Internet Group Management Protocol (IGMP).

Transport Protocols

Transport protocols are those that operate at the Transport layer of the TCP/IP model (and the Transport layer of the OSI model). These protocols and standards include:

  • Transmission Control Protocol (TCP): The most popular protocol stack used today to transmit and receive data throughout corporations and the Internet.
  • User Datagram Protocol (UDP): A connectionless transport protocol that is often used when faster service is required.

The following table describes these protocols in more detail.

Transport Protocol Description
TCP
  • A connection-oriented, guaranteed-delivery protocol used to send data segments between computers over a network such as the Internet.
  • Works with IP to transfer data.
  • Responsible for breaking up data into segments, reassembling them at the other end, resending data lost in transit, and sequencing data.
  • Sends data, waits for an acknowledgement, and fixes erroneous data.
  • Used in both LANs and WANs.
UDP
  • A connectionless, best-effort delivery protocol that is used with IP.
  • Lacks reliability, flow-control, and error-recovery functions.
  • Is less complex than TCP.
  • Provides faster service because it is a connectionless protocol.

Ports and Port Numbers

A Network port is a number that identifies one side of a connection between two computers. Because so many types of messages travel to and from network nodes, port numbers help to identify which process or application is sending or should receive a particular message.

Application Protocols

Application protocols are those that operate at the Application layer of the TCP/IP model (and the Transport layer of the OSI model). They are the primary means for an application to request network services from the operating system. These protocols and standards include:

  • Border Gateway Protocol (BGP)
  • Domain Name System (DNS)
  • Dynamic Host Configuration Protocol (DHCP)
  • File Transfer Protocol (FTP)
  • Gopher
  • HyperText Transfer Protocol (HTTP)
  • Network File System (NFS)
  • Network Time Protocol (NTP)
  • Post Office Protocol version 3 (POP3)
  • Routing Information Protocol (RIP)
  • Simple Mail Transfer Protocol (SMTP)
  • Simple Network Management Protocol (SNMP)
  • Telnet
