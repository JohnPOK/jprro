Describe The Most Popular Network Models

Topic D

Network Models

To ensure consistent network communications, hardware and software from various sources must be able to work together to transfer data to the intended recipient. Due to the complexity of the facets of data communications, it can be helpful to envision an abstract model to break down the overall process into more manageable units.

Layered Models

Anything that is complex can be easier to understand if you break it down into smaller components. Networking is no exception. All networks provide the same basic function of transferring data from sender to receiver, but each network can use different network hardware and software to accomplish this function. All of the hardware and software products must work together to successfully transfer data. By separating the data communication process into distinct layers, hardware and software vendors can develop products that address each layers responsibilities. As long as the interface between each layer does not change, the hardware and software within a layer can be updated or replaced without affecting the overall data communication process.

The two most popular network models are the Open System Interconnection (OSI) model and the TCP/IP model.

  • OSI Model: Developed by the International Organization for Standardization (ISO) for communication in open system networks. This model divides the data communication process into seven layers.
  • TCP/IP Model: Developed by the United States Department of Defense. It is a four-layer model.

The OSI Model

The OSI Model:

  • Reduces the complexity of networking to seven smaller, more manageable pieces.
  • Allows for the standardization of interfaces among devices.
  • Prevents changes in one layer from affecting other layers.
  • Enables engineers to concentrate on the specific design of functions in each layer .
  • Includes four lower layers that are mostly concerned with transferring data.
  • Includes three upper layers that deal with the applications running on the end devices.

Layers of the OSI Model

This is a standard visual representation of the OSI model:

The following table describes each layer.

Layer Description
Layer 1: Physical layer
  • Provide the means for transmitting data packets over a physical medium.
  • Specifies the elecrical and mechanical characteristics, such as voltage, frequency, and transmission medium on the network.
  • Receives fully formatted data packets from the Data Link layer and places them on the media.
  • NICs and APs operate at this layer.
  • Determines the mode and medium of data transmission, which are factors that affect transmission speeds.
  • At this layer, the data (protocol data unit, or PDU) is referred to as "bits" or bitstream.
  • Examples include: twisted pair, coax, fiber optic cable, baseband, broadband, clock rate, radio frequency, speed and bandwidth, RJ-45, BNC, AP, NIC, hub, repeater.
Layer 2: The Data Link layer
  • Responsible for transferring data between adjacent nodes.
    • Groups data into frames and attaches the address of the receiving node to each frame.
    • Adds error detection codes to frames and discards any damaged frames.
  • Most switches operate at this layer.
  • At this layer, the data is referred to as "frames".
  • There are two sub-layers: Logical Link Control (LLC) sublayer, which identifies the Layer 3 payload, and Media Access Control (MAC) sublayer, which adds a physical address to the frame.
  • Examples include: switch, bridge, Ethernet, frame-relay, PPP, PPPoE, ARP.
Layer 3: The Network layer
  • Responsible for addressing data packets, selecting the best route between a source and a destination, and routing the packets from a source to the destination through the network.
  • Routers and some switches operate at this layer.
  • Defines protocols for interconnecting two or more similar networks.
  • At this layer, the data is referred to as "packets".
  • Examples include: router, Layer 3 switch, firewall, IP, ICMP, IGMP.
Layer 4: The Transport layer
  • Ports are used as the endpoints for communication between network applications and services.
  • Accepts data from the upper layers, and breaks it into smaller units known as segments (TCP) or datagrams (UDP).
  • Supports connection-oriented (TCP) communication.
    • Establishes connections between devices and applications, maintains the connection, and then terminates.
    • Passes segments to the lower layers and ensures reliable delivery of all segments.
    • Provides flow-control through sliding windows to prevent overflowing receive buffers.
    • Adds a sequence number to each segment, which helps reconstruct the original sequence of segments and detect any missing segments.
    • Sends acknowledgements for received segments and retransmits any segments that are not acknowledged as being received.
    • Performs error correction by retransmitting any segments whose frames were damaged and discarded by the Data Link layer.
  • Supports connectionless (UDP) communication.
    • Best effort delivery - no guarantee of reliability.
    • No connection, no flow-control, no sequence numbers or acknowledgements, no error correction.
  • Examples include: TCP, UDP, firewalls, multilayer switches.
Layer 5: The Session layer
  • Controls how, when, and for how long a device can transmit or receive data.
  • Specifies procedures for the connection, termination, and re-establishment of sessions.
  • At this layer and above, the data is simply called "data".
  • Examples include: TCP and UDP ports, RPC (Remote Procedure Calls), NetBIOS, and named pipes.
Layer 6: The Presentation layer
  • Responsible for encoding data into a standard network-compatible format.
  • Uses standard data formats to enable devices with different representation techniques to communicate with each other.
  • Adds services such as data compression and encryption.
  • Examples include: ASCII, Unicode, encryption levels, JPEG, GIF, PNG, MOV, AVI, WMV, audio and video codecs.
Layer 7: The Application layer
  • Provides protocols that allow applications to access utilities and services on the network.
  • Defines protocols for tasks such as transferring files, sending emails, and saving data to a network server.
  • Advertises itself to the server resources available in each system for usage on the network.
  • The only layer with which the user directly interacts.
  • Examples include: HTTP/HTTPS, SMTP, IMAP4, POP3, SNMP, Telnet, SSH, SMB, NFS, FTP, TFTP, NTP, etc.

The TCP/IP Model

The TCP/IP model was:

  • Developed to allow the addition of new technologies and create a more flexible architecture, which can easily allow the modification of existing protocols.
  • Initially referred to as the DoD (or Department of Defense) model, but later it was renamed after two of its most important protocols: TCP and IP.
  • Note: An updated version of the TCP/IP model has five layers. This was accomplished by splitting the Network Interface Layer into the Data Link layer and Physical layer. Also, the Internet Layer is renamed to the Network layer in this updated TCP/IP model.

Layers of the TCP/IP Model

This is a standard representation of the TCP/IP model, but some of the layers' names have been changed over time.

The following table describes each layer.

Layer Description
Layer 1: The Network Interface layer
  • Provides services to send and receive data packets on the network.
  • Defines protocols for moving data frames between adjacent nodes, and for accessing the medium by the devices.
  • Defines the protocols for encoding and transmitting data over the network media.
Layer 2: The Network layer
  • Sometimes referred to as the Internet layer.
  • Provides addressing and routing services.
  • Involves transferring data from a source to a destination network when multiple networks are connected together.
Layer 3: The Transport layer
  • Sometimes referred to as the Host-to-Host layer.
  • Provides connection establishment and communication services.
  • Defines protocols for end-to-end transfer of data, along with error and flow controls.
  • Contains two important protocols - TCP and UDP.
Layer 4: The Application layer
  • Provides definition of protocols for file, email, and hypertext transfer.
  • Handles the encoding of data, controls the sessions, and defines socket services and other utilities over TCP/IP.

Network Model Comparison

Some of the similarities between the OSI and TCP/IP models include:

  • Both models have a similar architecture.
  • Both models have the equivalent of an Application, Transport, and Network layer.
  • Both models have their lowest layer connected to the physical network.

Some of the dissimilarities between the OSI and TCP/IP models include:

  • The OSI model was developed to standardize networking - The TCP/IP model was specifically developed to execute Internet-related tasks such as remote login, email, and resource sharing.
  • The OSI model has seven architectural layers - The TCP/IP model has only four layers and does not have a dedicated Session or Presentation layer.
  • In the TCP/IP model, the Application layer handles the responsibilities of the OSI model's Application, Presentation, and Session layers.
  • The OSI model is an open model that could support any network protocol, and as a result, the functionality of each layer is not clearly defined and optimized - In the TCP/IP model, the protocols define the functionality of each layer for optimal performance.
