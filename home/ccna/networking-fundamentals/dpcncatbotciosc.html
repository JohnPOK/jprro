Introduction to Cisco Networks and Cisco IOS Commands

Now that you have reviewed the common components of networks, it is time to focus on Cisco networking.

Common Cisco Network Components

The two most common Cisco network components are the switch and the router.

Switches:

  • Provide desktop access to the network at the Access layer.
  • Enable segmentation of the local area network (LAN) into virtual LANs (VLANs) for larger LANs.
  • Cisco Catalyst switches are special switches that provide several choices in interfaces, depending on the type of LAN. On Catalyst switches, light-emitting diodes (LEDs) enable you to monitor switch activity and performance:
    • The SYST LED displays the system status continuously.
    • The switch status is shown by the SYST LED color.
    • The status (STAT), duplex (DPLX), and speed (SPD) LEDs show information that is being displayed by the port status LEDs.
    • Press the Mode button to cycle the LEDs through the STAT, DPLX, and SPD displays.
  • In the traditional Cisco three-tier model, Layer 2 switches are used at the Access and Core layers, while Layer 3 switches are used at the Distribution layer.
  • In the collapsed core model, Layer 3 switches are used at the Distribution/Core layer.

Routers:

  • Routers are located at the Distribution layer of the three-tier hierarchical model.
  • Can be used to connect LANs and for WAN connections.
  • The larger the number of connections (e.g., to branch and remote offices), the larger the router needed to support these connections.
  • Cisco builds a wide array of router products with varying processor speeds, interface types and densities, and memory capacity.
  • Some router models share a commong interface, which makes upgrading easier and more cost effective.

Cisco Inernetworking Operating System (IOS)

Cisco Inernetworking Operating System (IOS) is a proprietary operating system that runs on most Cisco switches and routers.

  • It encompasses the operating system and application program of these devices.
  • It provides connectivity options for carrying traffic between devices.
  • It provides network protocol implementations to provide upper-layer and routing support.
  • It provides security options to control access to network use.
  • It provides scalability options to add capabilities as the network grows.
  • It provides reliability options for high availability and dependable network access.

Command Line Modes for Cisco IOS

To use the command line interface (CLI) for viewing device status or changing its configuration in Cisco IOS, you must be in the proper command-line mode to have the desired command available. Each device has three primary modes - user EXEC mode, privileged EXEC mode, and global configuration mode.

  • The first two modes are for checking the status of the device.
  • The third mode is for specifying configuration for the entire device.

In user EXEC mode, you can:

  • View statistics and basic router functions.
  • Perform basic management functions, such as clearing counters, resetting terminal lines, and so forth.
  • Perform only a subset of the available IOS commands.
  • Use basic troubleshooting commands such as ping, traceroute, and telnet.
  • Use show commands that give information about device interface, program, memory, and utilization.

The command-line prompt indicates that you are in user EXEC mode by showing the device name followed by the greater-than symbol (>). For instance, a generic router might have a prompt like router> to indicate user EXEC mode.

In privileged EXEC mode, you can:

  • Perform all user EXEC commands.
  • View or change configuration information.
  • Clear informational counters.
  • Manage the operating system.
  • Start the command-line editor and issue the command to move to global configuration mode.

In this mode, the command-line prompt shows the device name followed by a hash symbol (#). For instance, a generic router might have a prompt like router# to indicate privileged EXEC mode.

In global configuration mode, you can change the configuration as it pertains to the entire device. For example, the hostname of a device is configured in this mode. The command prompt in this mode includes (config)#. For instance, if you enter global configuration mode for router, the command prompt will say router(config)#.

Interface configuration mode is an example of using a global configuration mode. In interface configuration mode, you can change the configuration of a specific interface. The command prompt for this mode includes (config-if)#. For example, to enable an interface on router, you would want the command prompt to read something like router(config-if)#.

Basic IOS Commands

The following table describes some basic IOS commands.

Command Description
enable Enter privileged EXEC mode from user EXEC mode.
configure terminal Enter global configuration mode from privileged EXEC mode.
hostname <host_name> Configure a device's hostname in global configuration mode.
interface <type> <number> Enter interface configuration mode from global configuration mode.
shutdown Disable an interface in interface configuration mode.
no shutdown Enable an interface in interface configuration mode.
copy Enables you to copy system images and configuration files.
end Return to privileged EXEC mode from any configuration mode.
exit Return to the previous mode from any mode.

Show Commands

The show commands in Cisco IOS enable you to view the status of various Cisco devices. Some of the show commands are listed in the following table.

Command Description
show version Displays information about the system hardware, including the number of ports, the amount of memory, and the system's base Ethernet address.
show interfaces Displays statistics and information for all ports.
show interface FastEthernet 0/1 Displays statistics and information for a FastEthernet interface, where 0 is the slot on the device and 1 is the interface number.
show ip <keyword> Displays information about the switch's IP configuration. Use this command to verify that the switch has been set up correctly for remote management via Telnet, SNMP, or a web browser.
show running-config Displays the contents of the configuration file used by the switch, including all non-default information.
show startup-config Displays the contents of the configuration file that was run at device startup.
