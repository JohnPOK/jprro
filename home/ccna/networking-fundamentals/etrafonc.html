Explain The Role And Function Of Network Components

Topic B

Network Components

Now that you can recognize common network types, it's time to look closer at the components that make up networks.

Network Interface Cards (NICs)

A network interface card (NIC) is a device that serves as a connection between a computer or device and the network. NICs are available for both wired and wireless networks.

  • Note: NIC is also called a network adaptor or network card.

The transceiver is the part of a NIC that is responsible for data transmissions through the communication medium. It has both a transmitter and a receiver integrated into it, so it can receive and send data. Most modern transceivers are built into the NIC.

  • Note: These are typical representations of wired and wireless NICs, but you might also encounter NICs that do not look exactly like these. For instance, a wireless NIC for a PC might have two antennas (for dual-band communications), and some USB NICs also have one or two antennas.

Servers and Clients

A server is a network component or process that offers resources to other network nodes. Servers can be general purpose, or they can provide specific applications or services. They can be physical hardware devices, software applications, or processes running on a computer (even a laptop). A data center or server farm is a collection of servers housed in the same physical facility that are interconnected with each other to provide enhanced server functionality. The server farm is normally connected to a central management server that manages and monitors the operations being performed by the various servers in the farm.

A client is a network node that requests services from a server. From a networking perspective, the client always initiates the connection to the server. The server then accepts or rejects the incoming connection attempt. Servers will reject the connection if the client asks for a service that the server does not offer, or if there are permission requirements or other criteria that the client does not meet.

Endpoint Devices

An endpoint device is a network node that communicates with other nodes and receives resources from servers, but generally does not offer services to other nodes. Basically, it is any client device, including:

  • PCs, laptops, tablets, and smart phones
  • Input/output devices like cameras, printers, and scanners
  • Retail automation devices like order kiosks and point-of-sale (POS) terminals

Virtual Machines (VMs)

A virtual machine (VM) is a computing device that uses hardware or software, or both, to emulate a distinct physical computer system. Depending on the needs of an organization, VMs can be configured as servers or as clients.

VMs reside on host computers, and are sometimes referred to as guests, or guest OSs. They typically have their own IP and MAC address, as well as a computer name. VMs save space and power, and are easier to back up, manage, and clone than physical machines. A typical datacenter will have hundreds or even thousands of servers running VMs on a few high-powered physical hosts.

Switches

A switch is a network device that filters and forwards or floods frames, which carry data between systems, based on the destination MAC address of those frames. Most switches have 8, 12, 16, 24, or 48 ports. Every port in a switch is considered as a single collision domain.

Switches inspect the destination address of each frame before forwarding the frames to one or more ports for transmission. If the destination address is unknown, the frame will be flooded to all ports except the original port on which it was received. They can work with multiple pairs of ports simultaneously to improve performance by aggregating the ports into a single logical connection.

Switches are responsible for forwarding data from the source to the destination, but because they forward data frames only to nodes or segments to which they are addressed, the chances of collision is greatly reduced and network performance is increased.

More about Collision Domains

A collision domain is a network segment where data transmissions can collide with one another if they are sent at the same time. When this occurs, both transmissions are discarded and must be sent again.

Access Points (APs)

An access point (AP) is a networking device that enables wireless devices to connect to a wired network. They transmit and receive network frames by using RF signals. Some APs have built-in routers, while others simply bridge wireless connections to the wired network and require a router or switch for additional services.

Controllers

In networking, a controller is a hardware device or software application that manages the flow of information between network nodes. They can also manage the nodes themselves.

There are many types of controllers in network deployments. Two types of controllers you are likely to encounter in Cisco network deployments are Cisco DNA Center and WLC.

  • Cisco DNA Center provides centralized management, automation, analytics, and security. It is implemented as a combination of Cisco DNA Center appliance and the Cisco DNA software.
    • Note: Cisco DNA stands for Cisco Digital Network Architecture.
  • A wireless LAN controller (WLC) is a network device that manages configuration of APs and their firmware, including the implementation of wireless policies, management settings, and security policies and settings. They provide centralized management of wireless network communications while simplifying overall network management overhead and costs.

Firewalls

A firewall is a network security device that monitors network traffic, allowing or blocking traffic based on a set of security rules. Generally implemented between a private network and the Internet, firewalls can manage both incoming and outgoing data.

  • They can be configured to block suspicious or unsolicited incoming traffic while allowing incoming traffic that has been sent as responses to requests from internal hosts.
  • They use complex filtering algorithms to analyze incoming data based on destination and source addresses, port numbers, and data types.
  • Information about all packets, incoming and outgoing, can be saved to a log and used for network monitoring or hardening.

There are different types of firewalls to provide different levels of protection for a network.

Firewall Types

There are several categories of firewalls, including:

  • Packet Filters. These firewalls control access to a network by reading and analyzing the incoming and outgoing packets. Typical packet filters inspect packet headers to gather information. Based on the IP addresses, ports, and protocols of the source and destination nodes, each packet is dropped (denied access) or accepted (allowed access).
  • Stateful inspection firewalls. These firewalls track the state of permitted network requests sent through the firewall, storing details about those requests like the source and destination addresses, ports, and protocol so that any matching response traffic can be dynamically permitted back through the firewall without an explicit rule.
  • Proxy firewalls. These firewalls filter messages at the application level. Rather than allow the client to make a direct connection with the destination, they fetch the desired content on behalf of the client, saving a copy in case it is needed by other clients. They are often referred to as application firewalls.
  • Hybrid firewalls. Hybrid firewalls combine the capabilities of the other types of firewalls.
  • Next-generation firewalls (NGFWs). A NGFW is a network security device that goes beyond the capabilities of traditional firewalls by providing application awareness and control, integrated intrusion prevention, and cloud-based threat intelligence.

Intrusion Detection and Prevention Devices

An intrusion detection system (IDS) is a security mechanism that monitors traffic, logs suspicious activity, and issues alerts when suspicious activity is discovered.

An intrusion prevention system (IPS) is a security mechanism that not only monitors and logs suspicious network traffic, it also instructs the router or firewall to reject, quarantine, or block the traffic. IPS systems can also instruct a host to drop a TCP connection by sending a reset (RST) segment to the host.

Routers

A router is a networking device that forwards packets based on their destination addresses. Routers can connect different types of networks, as long as they use the same network level protocol. One or more metrics are configured to enable the router to determine the optimal path for data forwarding and delivery. They work with routable protocols, which provide separate network and node addresses. A router can be a dedicated device, incorporated into a multifunction device, or implemented as software running on a node that has two or more NICs.
