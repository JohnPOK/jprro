Cisco Discovery Protocol (CDP)

Cisco Discovery Protocol (CDP) is a proprietary Cisco protocol used to aid in the discovery and management of Cisco devices. CDP works at Layer 2 to provide information about the device to all directly connected neighbors.

Cisco routers and switches use CDP messages to advertise device information to neighbors through multicast addressing.

You can use CDP in the show command to view details about neighbors.

    show cdp neighbors [<type> <number>] show cdp neighbors detail show cdp entry <name>

CDPv2 is the latest version with advanced device tracking capabilities.

CDP is enabled by default on all Cisco devices. All Cisco devices that run CDP maintain a neighbor table. They add entries to this table by listening for the multicast SubNetwork Access Protocol (SNAP) frame from the directly attached neighboring devices. Once an entry is listed in the table, a timer is set for the entry, so devices that go down will be deleted from the table after a few minutes. By viewing the information in the neighbor table, you can find out how your devices are physically connected in the network topology.

Link Layer Discovery Protocol (LLDP)

Link Layer Discovery Protocol (LLDP) is an industry standard protocol that you can use to discover Cisco and non-Cisco devices. LLDP works as a Layer 2 protocol and is used by network devices to advertise their current status and capabilities.

Devices use LLDP attributes that include type-length-value entries (TLV) to advertise and solicit information from neighbors. TLVs include:

  • System description and capabilities
  • Device identity
  • Port description
  • Port VLAN
  • Configuration information

LLDP is defined by the Institute of Electrical and Electronics Engineers (IEEE) 802.1ab standard. Because it operates on the Data Link layer, LLDP allows discovery of devices running different network protocols. By default, LLDP is disabled on a device. Therefore, to use LLDP for device discovery, it must be enabled on the device and then on the interface. For each interface, LLDP can be used to discover one neighboring device.

  • Note: When LLDP is first enabled on a device, all interfaces will automatically send and receive LLDP frames.

Spanning Tree Protocol (STP)

Spanning Tree Protocol (STP) is an Institute of Electrical and Electronics Engineers (IEEE) 802.1D bridge and Layer 2 link-management protocol that prevents undesirable loops in the network by providing path redundancy. There can only be one existing active path between two stations for the proper functioning of a Layer 2 Ethernet network. STP operation is transparent to end stations that cannot detect whether they are connected to a switched LAN with multiple segments or to a single LAN segment.

The process for creating a loop-free STP topology is:

1. Elect the root bridge The root bridge or root switch is the reference point for all switches in the spanning tree topology. The root bridge has the lowest Bridge ID of all the switches. All other switches are referred to as non-root bridges.
  • Note: The Bridge ID consists of the Bridge Priority and the switch's MAC address. The default Bridge Priority is 32768. If all switches have the same Bridge Priority, the switch with the lowest MAC address will become the root bridge.
2. Select one root port on each non-root switch. Once a root bridge has been elected, each non-root bridge determines which of its ports has the lowest path cost to the root bridge, and that port is designated as the root port.
3. Select one designated port on each network segment. The port that advertises the lowest root path cost is elected as the designated port. All other ports are identified as non-designated or blocking ports.

Rapid STP

As its name implies, Rapid STP (RSTP) (IEEE 802.1w) provides faster recovery in response to network changes or failures, by using new convergence behaviors and bridge port roles. RSTP was designed to be backwards-compatible with STP.

Root Bridges

As mentioned previously, the root bridge is the central reference point in a spanning tree topology, so there is one root bridge for each VLAN. But what if that switch fails or becomes unavailable? You can designate a secondary root bridge by setting its Bridge Priority to the second-lowest value.

Port States

When the spanning-tree algorithm is run, it transitions each port through several different states to ensure that placing a port in forwarding mode will not cause a loop. The spanning-tree port states are:

  • Blocking: When you enable STP, an interface always starts in the blocking state. After the device initializes, it sends a Bridge Protocol Data Unit (BPDU) to the Ethernet and radio ports of the access points. When the access points exchange BPDUs, it helps in identifying the root bridge.
    When an interface is in a blocking state, it does not participate in frame forwarding, it discards the frames received on the port, and it does not learn addresses. It does, however receive BPDUs. Ports connected to redundant links are put in a blocked state for an indefinite period. If the original link goes down, then the blocked port is unblocked and used in its place.
  • Listening: The listening state is the first interface state after the blocking state. When STP determines that the interface should participate in frame forwarding, the device enters this state.
    When an interface is in the listening state, it discards frames received on the port and does not learn addresses. It does, however receive BPDUs. The default duration of the listening state is 15 seconds.
  • Learning: The learning state is the next interface state after the listening state. When an interface is in the learning state, it prepares to participate in frame forwarding. In the learning state, the interface discards frames received on the port, but it will learn addresses, update MAC address tables, and receive BPDUs. The default duration of the learning state is 15 seconds.
  • Forwarding: The forwarding state is the next interface state after the learning state. When an interface is in the forwarding state, it participates in frame forwarding. It receives and forwards frames recieved on the port, and it learns addresses and receives BPDUs.
  • Disabled: The disabled state is the next interface state after the forwarding state. When an interface is in the disabled state, it does not participate in frame forwarding or in the spanning tree and is non-operational. It discards frames received on the port. The interface in disabled state does not learn addresses or receive BPDUs.

PortFast

STP PortFast causes a Layer 2, LAN port, configured as an access port to enter forwarding state immediately, by allowing the device to bypass the listening and learning states.

Interfaces connected to a single workstation or server should not receive BPDUs. A PortFast-enabled port can immediately transition to the blocking state if necessary (this could happen on receipt of a superior BPDU). PortFast can be enabled on trunk ports and can have an operational value that is different from the configured value.

Link Aggregation Control Protocol (LACP)

An EtherChannel is a port-link aggregation technology that combines multiple physical Ethernet links into one logical link, providing fault tolerance and high-speed links for connecting switches, routers, and servers.

Link Aggregation Control Protocol (LACP) is an IEEE 802.3ad standard that supports the automatic creation of EtherChannels by permitting the exchange of LACP packets between LAN ports that are in passive and active modes. The protocol dynamically learns about the capabilities of LAN port groups and informs the other LAN ports.

When LACP identifies the correctly matched Ethernet links, it allows the grouping of links into an EtherChannel. Then, the EtherChannel is added to the spanning tree as a single bridge port.

Using criteria such as port speed and trunking state, both passive and active modes allow LACP to negotiate between LAN ports to determine if an EtherChannel can be formed. Layer 2 EtherChannels also use VLAN numbers as a criterion.

Even LAN ports that are in different LACP modes can form an EtherChannel as long as the modes are compatible. For Example:

  • Two LAN ports that are in active mode can form an EtherChannel.
  • A LAN port that is in active mode can form an EtherChannel with another LAN port that is in passive mode.
  • Two LAN ports that are in passive mode cannot form an EtherChannel because neither of the ports will be able to initiate a negotiation.

System Configuration Dialog

The System Configuration Dialog is an interactive mode to perform the initial steps for switch configuration. It appears automatically when a switch is powered on for the first time or when the Non-volatile RAM (NVRAM) does not contain a configuration file for the switch.

Cisco IOS enters into switch setup mode and prompts the user to enter basic switch configuration information, accepting user input for the required configuration settings. It also requests confirmation to save the settings. Once confirmed, IOS saves the configuration file into NVRAM and uses it as the startup-config file.

The System Configuration Dialog is used by administrators to quickly enter basic switch configuration information. It is also used when an administrator decides to erase the startup-config file in the NVRAM and create a new config file. The prompts that appear in the System Configuration Dialog depend on the hardware and software installed on the switch. The System Configuration Dialog cannot be used to configure advanced switch settings.

You can access help resources at every prompt of the System Configuration Dialog by entering a question mark (?). At any point during the setup, you can press Ctrl+C to stop the configuration steps without saving the changes and enter privileged EXEC mode. By entering the setup command in privileged EXEC mode, you can configure additional switch settings.

Using the System Configuration Dialog for Initial Switch Configurations

The following commands are used in the Using the System Configuration Dialog for Initial Switch Configurations lab:

  • copy running-config startup-config: Saves the configuration file to NVRAM.
  • enable: enters privileged EXEC mode.
  • end: Ends and exits configuration mode.
  • erase startup-config: Deletes the startup configuration file in NVRAM.
  • interface <type> <number>: Changes from global configuration mode to interface configuration mode.
  • ip default-gateway <ip-address>: Configures a default gateway IP address to which traffic destined to remote networks has to be forwarded.
  • ping <ip-address>: Sends an Internet Control Message Protocol (ICMP) echo request to the specified address.
  • reload: Reboots the switch.
  • show running-config: Displays the active configuration file.
  • show startup-config: Displays the backup configuration file.
  • [no] shutdown: Disables an interface. The no form enables an interface.
