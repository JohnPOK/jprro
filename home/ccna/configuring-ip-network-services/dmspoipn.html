Describe Major Services Provided on IP Networks

Topic A

IP Network Services

There are a wealth of services available for implementation on an IP network. You will start investigating these by looking at some file transfer services.

Network Services.

Common network services include:

  • File-transfer services such as File Transfer Protocol (FTP) and Trivial FTP (TFTP).
  • Dynamic Host Configuration Protocol (DHCP): A network service that automatically assigns IP addresses and other TCP/IP configuration information on network nodes configured as DHCP clients.
  • Network Address Translation (NAT).
  • Network Time Protocol (NTP).
  • Domain Name System (DNS): A Tranmission Control Protocol/Internet Protocol (TCP/IP) name resolution service that translates Fully Qualified Domain Names (FQDNs) into IP addresses.
  • Network Management.

FTP

The File Transfer Protocol (FTP) is a standard networking protocol that moves files between network nodes. Because it uses a client-server model, two connections are required: one for moving the data and one for control purposes

FTP is not encrypted, which might pose a security risk. Secure versions are available:

  • FTPS: FTP with TLS/SSL security uses TLS or SSL to encrypt all FTP communications.
  • SFTP: SSH FTP uses SSH to encrypt FTP data transfers.

This table lists the default transport protocols and ports for each type of FTP.

FTP Type Transport Protocol and Port Number
FTP TCP port 20 for data connection and TCP port 21 for control information.
FTPS
  • TCP or UDP ports 989 and 990 for implicit FTPS.
  • TCP ports 20 and 21 for explicit FTPS.
SFTP TCP or UDP port 22 (the same port number as SSH).
Implicit and Explicit FTPS

FTPS provides implicit and explicit modes.

  • With implicit FTPS, an SSL connection is established first, then login and file transfer can occur.
  • With explicit FTPS, a traditional FTP connection is established, and then a secure SSL connection is established prior to login. FTPS clients send the AUTH command prior to logging in so that credentials are secured.

Explicit FTPS is the preferred mode, but there are still implicit FTPS clients in use.

TFTP

The Trivial File Transfer Protocol (TFTP) is an Application layer protocol that enables the transfer of configuration files between Cisco devices. TFTP is a simple version of File Transfer Protocol (FTP). It uses User Datagram Protocol (UDP) as the transport protocol for providing basic file transfer features in a network. TFTP uses UDP port 69.

Many Cisco devices, most commonly Cisco IP phones, receive configuration files from a TFTP server. The tftp option is used along with the copy command to move configuration files between a switch's or router's Random Access Memory or Non-volatile RAM (NVRAM) and a TFTP server. On the NVRAM or TFTP server, when a file is copied, the existing file is erased and replaced by the new file. However, when a configuration file is copied into the running config file in the RAM, the copied file is merged with the existing running config file. The command to copy configuration files is:

copy tftp {running-config | startup-config | device:filename}

Cisco IP phone devices are not pre-configured with the IP address of the TFTP server. They need to obtain this IP address from the DHCP server. Such devices use option 66 of the dhcp command to request the DHCP server for the IP address of the TFTP server. If the network has two TFTP servers, devices can use option 150 to obtain the IP addresses of the primary and secondary TFTP servers.

Command Reference: TFTP Commands

This section contains the commands you will use in the next set of labs.

TFTP Commands

The following commands are used in the TFTP labs:

  • configure terminal: Enter global configuration mode from privileged EXEC mode.
  • copy running-config [startup-config | tftp | <device>:<filename>]: Copies a device's running configuration file to another destination.
  • copy tftp [running-config | startup-config | <device>:<filename>]: Copies a file from a TFTP server to another destination.
  • enable: Enter privileged EXEC mode.
  • end: Ends and exits configuration mode.
  • erase startup-config: Deletes the startup configuration file in NVRAM.
  • exit: Exits one level in the menu structure.
  • hostname <host-name>: Sets the device name.
  • interface <type> <number>: Changes from global configuration mode to interface configuration mode.
  • interface /dg <type> <number>: Used in NetSim to assign a default gateway IP address to a workstation interface.
  • ip address <ip-address> <subnet-mask>: Assigns an IP address to an interface.
  • ipconfig /ip <ip-address> <subnetmask>: Used in NetSim to assign an IP address and subnet mask to a workstation interface.
  • [no] shutdown: Disables an interface. The no form enables an interface.
  • ping <ip-address>: Sends an Internet Control Message Protocol (ICMP) echo request to the specified address.
  • show running-config: Displays the active configuration file.
  • show startup-config: Displays the backup configuration file.
  • show tftp-configs: Used in NetSim to display a list of files copied using TFTP and stored on a PC.
