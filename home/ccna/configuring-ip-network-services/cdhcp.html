Configure DHCP

Another popular network service is automatic IP address assignments for clients. Because IP addresses must be unique, having them dynamically assigned ensures that no two client devices will ever have the same IP address.

DHCP

The Dynamic Host Configuration Protocol (DHCP) is a protocol that automates the assignment of IP addresses, subnet masks, gateways, and other IP parameters. You configure the DHCP server with the information it needs to provide to clients, including:

  • Subnet IDs and masks to represent the IP addresses that are available in the subnet. This is called the DHCP scope.
  • The IP address of the default router on the subnet.
  • The IP addresses of the DNS servers.
  • The domain name.
  • Reserved/excluded addresses that are not available for lease.
  • Additional options for speciality services.

DHCP uses TCP/UDP ports 67 (server) and 68 (client).

The DORA Process

The entire process of communication between a DHCP server and a host/DHCP client is called the Discover Offer Request Acknowledge (DORA) process.

1. A host sends a DHCP Discover message to communicate with a DHCP server. Because hosts do not have any initial IP settings to start communicating in the network, they use the addresses 0.0.0.0 and 255.255.255.255. For example, a host that does not have an IP address can use the IP address 0.0.0.0 as its source IP address to send discover messages to the DHCP server. It can use the IP address 255.255.255.255 to broadcast the discover message within a subnet.
2. The DHCP server sends a DHCP Offer message providing its IP address to the DHCP client.
3. When the DHCP client receives an offer message, it sends a DHCP Request back to the DHCP server. When multiple DHCP servers send offer messages to the client, the client responds to the first offer message.
4. When the DHCP server receives the request message, it sends a DHCP Acknowledgement message, along with the required IP settings. The IP address is usually leased to the client for a specific time period.

DHCP Servers

The DHCP server is configured with a list of IP addresses called a DHCP scope. The scope contains a range of IP addresses and a subnet mask, and it can contain other options, such as a default gateway and DNS server addresses. A scope also needs to specify the duration of the lease. When the lease period is over, the client needs to renew the lease with the DHCP server.

When a DHCP server enables the scope, it automatically leases TCP/IP information to DHCP clients for the established lease duration. The DHCP server determines the lease duration, which can be a defined time period or an unlimited length of time.

DHCP Relays

In most enterprise networks, DHCP servers are centralized and serve hosts in many different remote subnets. Routers in the subnets use a mechanism called DHCP relay to route DHCP messages between the hosts that need IP address assignment and the DHCP server.

The routers in a network that enable communication between a central DHCP server and hosts in different subnets are called DHCP relay agents. Relay agents take the place of a DHCP server on a network segment. Clients do not realize they are communicating with a relay agent. The relay agent uses broadcasting to communicate with the clients, but uni-casting when communicating with the DHCP server.

The ip helper-address <server-ip> subcommand is configured on each router interface that will receive broadcast DHCP Discover messages from clients to relay DHCP messages correctly to the specified DHCP server address.

Command Reference: DHCP Configuration

This section contains the commands you will use in the next set of labs.

DHCP Configuration

The following commands are used in the DHCP Configuration labs:

  • configure terminal: Enters global configuration mode from privileged EXEC mode.
  • default-router <address>: Specifies the IP address of a default gateway for DHCP clients.
  • dns-server: Specifies the DNS OP servers available to a DHCP client.
  • domain-name <domain>: Specifies the domain name for a DHCP client.
  • enable: Enters privileged EXEC mode.
  • end: Ends and exits configuration mode.
  • exit: Exits one level in the menu structure.
  • interface <type> <number>: Changes from global configuration mode to interface configuration mode.
  • ip address dhcp: Configures a router as a DHCP client.
  • ipconfig /all: Used in NetSim to display the IP addresses and MAC addresses on a workstation.
  • ipconfig /ip dhcp: Used in NetSim to configure a workstation to obtain IP addressing information from a DHCP server.
  • ip dhcp excluded-address <ip-address> <last-ip-address>: Configures a range of excluded addresses.
  • ip dhcp pool <pool-name>: Creates an IP address pool.
  • lease <days> <hours> <minutes> | infinite: Configures a custom lease duration for DHCP.
  • network <ip-address> <subnet-mask>: Enables a DHCP server for a particular network segment.
  • [no] service dhcp: Configures the router to respond to any DHCP requests. The no form disables the DHCP service and prevents a DHCP relay agent from forwarding DHCP requests.
  • show dhcp lease: Displays the lease duration of the IP address.
  • show dhcp server: Displays IP configuration information contained in DHCPOFFER packets.
  • show ip dhcp binding: Displays active DHCP-assigned IP addresses.
  • show ip dhcp pool: Displays a basic summary of a configured DHCP pool.
  • show ip dhcp server statistics: Displays general operating statistics for a DHCP server process.
  • show ip interface brief: Displays a brief summary of interface status and configuration.
  • show ip route: Displays the IP routing table.
  • show running-config: Displays the active configuration file.
