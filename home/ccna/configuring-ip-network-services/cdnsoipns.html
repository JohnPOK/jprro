Configure DNS on IP Networks

Topic E

Configure DNS

The ability to translate IP addresses into meaningful names makes it much easier for humans to remember them. The primary way of accomplishing this on an IP network is to implement Domain Name System (DNS).

Host Names

Host names are labels assigned to network devices and endpoints. Host names are much easier to remember than IP or MAC addresses, and they can be configured to be quite descriptive.

The HOSTS File

A host table is a mapping of host names to the associated IP addresses. They are unique names assigned to IP hosts that provide an easy way to configure and manage remote systems within the network. Host tables are generally stored in a text file called HOSTS (no matter what operating system is running).

A host table provides static host name to IP address mapping on a router. Host tables can be created on any router on the network that requires one, and they are valid only on the router in which they are built. You use the ip host global configuration command to create a host table. To add a host entry to an existing table, use the ip host command.

Domain Name System (DNS)

Domain Name System (DNS) is a TCP/IP name resolution service that translates a Fully Qualified Domain Name (FQDN), which is a domain name of a specific location in the DNS hierarchy, into an IP address (or vice versa). A domain is a group of computers whose communication is managed as one unit. A domain name is a unique name that identifies a domain. DNS consists of a system of hierarchical databases that are stored on separate DNS servers on all networks that connect to the Internet. DNS servers do the following:

  • List IP addresses and related computer names.
  • Store, maintain, and update databases.
  • Use the databases to respond to DNS client name-resolution requests to translate host names into IP addresses.
  • Work together to resolve FQDNs.
  • On internal networks, a local DNS service can resolve host names without using external DNS servers.

A host name is the unique name given to a network node on a TCP/IP network. The host name combined with the host's domain name forms the node's FQDN. The domain name space consists of information about the hierarchy of domains and the hosts under each domain, which is referred to by the DNS servers for mapping domains. The FQDN denotes the specific location of the host in the DNS hierarchy from the top-level domain to the last.

DNS uses TCP/UDP port 53.

Name Resolution with DNS

There are two types of DNS servers: authoritative and recursive.

  • An authoritative DNS server provides the IP mapping to a host name when requested by a recursive DNS server.
  • A recursive DNS server gathers the name resolution requests from hosts and is responsible for fulfilling those requests.

DNS lookup is the process of resolving the IP address for a host name. This operation includes the following steps:

1. A client sends a name resolution request to its preferred DNS server.
2. The preferred DNS server checks its DNS cache entries or its local DNS database and returns the IP address to the client. If the DNS server does not find a match:
a. It requests the root server for the IP address of the DNS server, which is responsible for the top-level domain.
b. The root server returns the IP address of the appropriate DNS server.
c. The preferred DNS server contacts the top-level domain's DNS server to resolve the name. If solved, the preferred DNS server receives the required IP address. If the top-level domain cannot resolve the name because of additional levels in the FQDN, it sends the IP address of the second-level DNS server.
3. The communication between DNS servers continues until it reaches the level in the DNS hierarchy where a DNS server can resolve the host name.
4. The preferred DNS server provides the client with the IP address of the target host.

Configuring DNS

The following commands are used in the Configuring DNS labs:

  • bedit <filename>: Used in NetSim to open a file from a workstation.
  • configure terminal: Enters global configuration mode from privileged EXEC mode.
  • enable: Enters privileged EXEC mode.

  • ipconfig /dns <ip-address>: Used in NetSim to set the primary DNS server for a workstation.

  • ip name-server <ip-address>: Configures the name server for a device.
