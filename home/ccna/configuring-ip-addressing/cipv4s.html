Configure IPv4 Subnets

Now that you have configured basic IP addresses, it is time to look at one of the most common ways to provide flexibility in addressing and more efficient address management.

Subnetting

Subnetting is a process that subdivides the IPv4 address space of a network into smaller groups of consecutive IP addresses. An IP subnet refers to a subdivided network. IPv4 subnetting methods create small logical networks within any IP network.

Implementing subnets enable more efficient use of the IP address space. For example, you can divide the 132.9.0.0/16 network into four subnets with network addresses such as 132.9.0.0/18, 132.9.1.0/18, 132.9.2.0/18, and 132.9.3.0/18 to prevent wasting IP addresses.

The conventional addressing techniques specifies IP addresses with two hierarchical components, namely network ID and host ID. However, in Cisco subnet addressing, the host portion of the address is further divided into subnet ID and host ID. Therefore, Cisco subnet addressing is designed with three hierarchical levels: network ID, subnet ID, and host ID. Subnetting borrows bits from the host portion and uses them to create subnets.

Subnetworks provide better manageability, performance, security for hosts and networks, and reduced network congestion.

Subnet Masks

A subnet mask is a 32-bit number that divides an IPv4 address into two parts:

  • the network part
  • the host part

It comprises a series of binary 1s that end abruptly, followed by all 0s. The dividing line between the 1s and 0s is also the dividing line between the network part and the host part of the IP address. A subnet mask uses the Boolean AND operation to remove the node ID from the IP address, leaving just the network portion.

The number of addresses on a subnet are determined by the number of binary 0s in the host portion of the subnet mask. To calculate how many host addresses are available, use the formula 2n - 2, where n is the number of 0s on the right side of the subnet mask.

Hosts use the subnet mask to determine if a packet's destination is on the same network or on a different network. If it is on the same network, the sender delivers the packet directly to the recipient. If the destination is on a different network, the sender delivers the packet to its default gateway. The router then assumes responsibility for delivery.

Default Subnet Masks

In legacy IP addressing, addresses were divided into address classes that had default subnet masks assigned to each class. This was called classful addressing, which has been deprecated and has been replaced by Classless Inter Domain Routing (CIDR). A basic understanding of the original IP addressing design is necessary to recognize the reasons for many current practices and technologies used to extend the useful life of IPv4.

Classful addresses were assigned a default subnet mask that was assumed by a receiving router or host based on the most significant bits in the IP address.

A default subnet mask always has all 1s or all 0s in an octet. Groups of IP addresses have specific default subnet masks based on the range of values of the first octet of the IP address.

  • In Class A, B, and C addresses, the network ID is divided from the host ID based on octet boundaries. Each default subnet mask always has a 255 or 0 when expressed in decimal notation.
  • Classes D and E do not have subnet masks.

This table lists the default subnet masks for class A, B, and C, along with the corresponding values in the first octet.

Default Subnet Mask Value of First Octet Network Class
255.0.0.0 0 to 127 Class A (/8)
255.255.0.0 128 to 191 Class B (/16)
255.255.255.0 192 to 223 Class C (/24)

In the Network Class column, the slash (CIDR) notation indicates how many bits are populated with 1s.

Subnet Mask Rules and Application

To conform to TCP/IP standards, subnets must follow a set of rules:

  • The 1s in the mask always starts at bit 32, at the left end of the mask.
  • The 0s in the mask always starts at bit 1, at the right end of the mask.
  • The 1s in the mask must be contiguous, with no 0s interspersed between them.
  • The only decimal values permitted in an octet are: 255, 254, 252, 248, 240, 224, 192, 128, and 0.
  • Each subnet must have its own unique network ID.
  • All devices on a subnet must share the same network ID.
  • All devices on a subnet must have the same subnet mask.
  • Nodes on a local subnet must have unique node IDs.

Applying a subnet mask involves a sequence of steps to isolate the network portion of the IP address from the host portion:

1. Convert both the IP address and the subnet mask to binary format.
2. Use the AND operation on the two binary numbers. In the AND operation, the 0s in the subnet mask convert all the bits in the node portion of the IP address to 0s, leaving the network portion of the address intact.

The Subnetting Process

Here are the stages in the subnetting process:

1. Identify the number of subnets required in the network.
2. Identify the number of hosts that need to be accommodated per subnet.
3. Starting from the address mask assigned to you by an addressing authority, select a subnet mask that will create the desired number of subnets while still leaving the required number of hosts.
When one subnet mask is applied to all the subnets, the same number of hosts can be accommodated in each subnet.
4. Determine the subnet IDs and host IDs.

For example, an organization has a network address of 192.173.28.0 with a subnet mask of 255.255.255.0, and it wants to set up six subnets to segregate traffic across six departments. To do this, three higher order bits in the last octet can be reserved for subnetting. The result is a subnet mask 255.255.255.224, or 8 subnets, each capable of supporting 30 hosts.

The general process is:

1. Identify the existing subnet mask.
2. Determine the number of bits to add to or remove from the mask.
3. Change the mask and identify the delta.
4. Calculate the subnet IDs and address ranges.

Variable Length Subnet Masks (VLSMs)

A variable length subnet mask (VLSM) is used to create subnets containing different numbers of hosts. In the VLSM process, you apply a custom subnet mask to the IP addresses and provide the number of hosts required for each subnet.

A network is first divided into subnets, then some of the subnets are divided into sub-subnets, and some of these are further divided. The detailed structure of routing information for one subnet group is hidden from routers in another subnet group.

For instance, a network with a prefix length of /24 (255.255.255.0) might contain three subnets: subnet 1 with 5 hosts, subnet 2 with 12 hosts, and subnet 3 with 28 hosts. By applying the variable subnet mask /29 (255.255.255.248) to subnet 1, /28 (255.255.255.240) to subnet 2, and /27 (255.255.255.224) to subnet 3, you waste only one IP address on subnet 1, and two IP addresses on each of subnets 2 and 3.

VLSM enables optimal use of the available address space in a network.

The Address Optimization Process

The steps to apply VLSM to optimize IP addressing in subnets are:

1. Determine the number of subnets required and the number of hosts to be accommodated in each subnet.
2. Begin VLSM subnetting with the subnet that has the highest requirement of hosts.
3. Determine the number of bits required in the subnet mask and generate the required number of host addresses.
4. Continue subnetting for the next highest subnet, and in this way, complete the VLSM.

For example, consider a network with the address space 173.35.4.0/24. The network requires three LAN subnets and one wide area network (WAN) subnet. LAN 1 requires 60 hosts, LAN 2 requires 28 hosts, and LAN 3 requires 12 hosts.

1. Since LAN 1 has the highest requirement, begin subnetting for LAN 1. Use 6 bits to get 26 - 2 = 62 host addresses. The prefix /26 moves the mask down the delta position 64, giving you just enough IP addresses for your case.
  • The delta is 64 - subnet IDs will increment by 64
  • Delta 64 gives you 4 subnets with a /26 mask:
    • 173.35.4.0
    • 173.35.4.64
    • 173.35.4.128
    • 173.35.4.192
  • Subnet 0 can be used for LAN 1:
    • 173.35.4.0 = subnet ID
    • 173.35.4.1 = first legitimate host IP
    • 173.35.4.62 = last legitimate host IP
    • 173.35.4.63 = broadcast
    Addresses 173.35.4.1/26 through 173.35.4.62/26 are available for hosts.
2. For LAN 2, you need addresses for 28 hosts:
  • 5 bits are needed to generate 25 - 2 = 30 host addresses.
  • You don't want these addresses to overlap with LAN 1, so you start with the next subnet 173.35.4.64 (subnet "64").
  • The available host range is 173.35.4.65 - 173.35.4.126.
  • If you use all of subnet 64, you can have 62 hosts, but you don't need that many.
  • You can move the mask down one more bit to /27 to further divide subnet 64 in half:
  • 173.35.4.64/27
    • 173.35.4.64 = subnet ID
    • 173.35.4.65 = first legitimate host IP
    • 173.35.4.94 = last legitimate host IP
    • 173.35.4.95 = broadcast
  • 173.35.4.96/27
    • 173.35.4.96 = subnet ID
    • 173.35.4.97 = first legitimate host IP
    • 173.35.4.126 = last legitimate host IP
    • 173.35.4.127 = broadcast (notice how the next IP would take you into subnet 128)
  • Each of these subnets gives you 30 usable hosts, which is perfect for your scenario.
  • 173.35.4.64/27 gives you a host range 173.35.4.65 - 173.35.4.94
3. Similarly, for LAN 3, 4 bits are needed to generate 24 - 2 = 14 host addresses.
  • You have subnet 96 available for use.
  • You don't need all 30 IP addresses from subnet 96, so can even further subdivide it.
  • Move the mask down one more bit to /28. It will split subnet 96 in half:
  • 173.35.4.96/28
    • 173.35.4.96 = subnet ID
    • 173.35.4.97 = first legitimate host IP
    • 173.35.4.110 = last legitimate host IP
    • 173.35.4.111 = broadcast
  • 173.35.4.112/28
    • 173.35.4.112 = subnet ID
    • 173.35.4.113 = first legitimate host IP
    • 173.35.4.126 = last legitimate host IP
    • 173.35.4.127 = broadcast (notice how the next IP would take you into subnet 128)
  • You only need to use one of theses subnets to supply 12 hosts, so use 173.35.4.96/28.
  • The prefix /28 can be used to generate host addresses 173.35.4.97/28 through 173.35.4.110/28
4. Continue subnetting for the next highest subnet, and in this way, complete the VLSM.

Command Reference: VLSM Practice

This section contains the commands you will use in the next lab.

VLSM Practice

The following commands are used in the VLSM Practice labs:

  • configure terminal: Enter global configuration mode from privileged EXEC mode.
  • enable: Enter privileged EXEC mode.
  • end: Ends and exits configuration mode.
  • exit: Exits one level in the menu structure.
  • interface <type> <number> <subinterface-id>: Creates a subinterface.
  • ip address <ip-address> <subnet-mask>: Assigns an IP address to an interface.
  • ipconfig /dg <ip-address>: Used in NetSim to assign a default gateway IP address to a workstation interface.
  • ipconfig /ip <ip-address> <subnetmask>: Used in NetSim to assign an IP address and subnet mask to a workstation interface.
  • ping <ip-address>: Sends an Internet Control Message Protocol (ICMP) echo request to the specified address.
  • show running-config: Displays the active configuration file.
