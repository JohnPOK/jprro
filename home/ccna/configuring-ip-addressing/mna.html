Static and Dynamic Addresses

One of the facets of managing IP addresses is to decide when to manually assign addresses and when to configure automatic (dynamic) addressing. There are advantages and disadvantages to both, depending on the situation. For instance, manually specifying IP addresses can be time-consuming and prone to error, but it is the preferred method of configuring servers and other devices that require consistent access.

Client OS IP Parameters

When you are managing addres assignment on client computers, you need to know where to check and configure addressing for each type of client.

  • In Windows, you can use the command ipconfig /all to view the current IP configuration, or you can use the Settings->Network & Internet app.
  • In Mac OS X 10.4, to view the IP configuration:
1. Select the Apple menu, then select System Preferences
2. Select Network, and in the Show list box, select the interface you want to view.
3. Select the TCP/IP tab to view IP configuration information.

  • In Linux, you can use the ifconfig command to view IP information.

Address Assignments for Switches and Routers

Routers and Layer 3 switches require a valid IP configuration for each interface that is configured on each subnet or VLAN. This IP address is typically used for configuring a default gateway for clients on the subnet or VLAN. You might want to develop a system for assigning addresses to these devices so that the addresses are easily recognizable as belonging to routers and switches as opposed to endpoint devices. It's likely that you will want to use static addresses for these devices.

Routers, Layer 3 switches, and Layer 2 switches should also be configured with a static IP address for management. This is typically a different subnet or VLAN than clients are connected to, and firewall rules or ACLs will restrict traffic to these addresses to administrators only.

On routers and Layer 3 switches, you can also assign IP addresses to loopback interfaces. A loopback interface is a software interface that emulates a physical interface and is useful because this interface never goes down and can be used by routing protocols for router identification with OSPF, for example.

Command Reference: IP Address Configuration on Switches and Routers

This section contains the commands you will use in the next set of labs.

IP Addresses on Switches and Routers

The following commands are used in the IP Addresses on a Switch and IP Addressing on a Router labs:

  • clock rate <clock-rate>: Sets the clock rate for a Data Communications Equipment (DCE) interface.
  • configure terminal: Enters global configuration mode from privileged EXEC mode.
  • enable: Enters privileged EXEC mode.
  • end: Ends and exits configuration mode.
  • hostname <host-name>: Sets the device name.
  • interface <type> <number>: Changes from global configuration mode to interface configuration mode.
  • ip address <ip-address> <subnet-mask>: Assigns an IP address to an interface.
  • ipconfig /dg <ip-address>: Used in NetSim to assign a default gateway IP address to a workstation interface.
  • ipconfig /ip <ip-address> <subnetmask>: Used in NetSim to assign an IP address and subnet mask to a workstation interface.
  • [no] shutdown: Disables an interface. The no form enables an interface.
  • ping <ip-address>: Sends an Internet Control Message Protocol (ICMP) echo request to the specified address.
  • show running-config: Displays the active configuration file.
