Configure IPv4 Addresses

The IPv4 protocol is the most widely used addressing and routing protocol in use today. The ability to configure IP addresses is paramount in ensuring that the data moves to its intended destination.

IPv4 Address Structure

  • Note: In this course, when the term "IP" is used, it is meant to refer to IPv4. IPv6 will always be written as IPv6.

Every network interface needs an IPv4 or IPv6 address to send and receive packets. An IPv4 address is a 32-bit number that is often represented in Dotted-Decimal Notation (DDN). When you use DDN, you can convert each group of 8-bits (octet) in an IP address from its binary form to decimal form and separate the resulting numbers by dots. The decimal range of each octet is from 0 to 255. An example of an IP address written in DDN notation is 192.14.56.108.

IPv4 Address Classes

The IPv4 address space is subdivided into classes based on the range of values in the network portion of the IP addresses. Classes define the type of addressing used.

  • Class A: Network addresses from 1 to 126
  • Class B: Network addresses from 128 to 191
  • Class C: Network addresses from 192 to 223
  • Class D: Network addresses from 224 to 239 are designated as multicast addresses
  • Class E: Network addresses from 240 to 255 are experimental and reserved addresses
  • Note: An IP network of Class A, B, or C is called a classful network. Class A networks are large networks, Class B networks are medium-sized networks, and Class C are small networks. These networks are defined by the class rules of IPv4 addressing. Classful addressing for hosts has been replaced with Classless Inter-Domain Routing (CIDR), but knowing the address classes can help you understand many concepts about IP addressing.

Address Classes and Binary

Address classes are actually defined by the leading bits (first binary numbers) found in the first (far left) octet. The following shows the first octet range of binary and dotted decimal numbers for each class.

  • Class A starts with 0
    • 00000000 = 0
    • 01111111 = 127
  • Class B starts with 10
    • 10000000 = 128
    • 10111111 = 191
  • Class C starts with 110
    • 11000000 = 192
    • 11011111 = 223
  • Class D starts with 1110
    • 11100000 = 224
    • 11101111 = 239
  • Class E starts with 1111
    • 11110000 = 240
    • 11111111 = 255

Unicast Addresses

A unicast address is an IP address that is used to communicate information from one node in a network to another node. Packets of data are sent from a source address to a particular destination address in the network. Other nodes in the network that do not match the destination unicast address ignore the data transmission.

Unicast addressing is predominantly used in LANs and the Internet.

Popular applications that use unicast addressing include Hyper-Text Transfer Protocol (HTTP), Simple Mail Transfer Protocol (SMTP), and File Transfer Protocol (FTP).

Broadcast Addresses

A broadcast address is an IP address that is used to communicate information from one node to all other nodes in a network. Packets of data are sent from a source address to every other node in the network. The IPv4 global broadcast address is 255.255.255.255. A broadcast that is specific to a particular network (directed broadcast) will have all host bits in the address set to binary 1s. For example, the directed broadcast for the network 192.168.1.0/24 is 192.168.1.255.

  • Note: On an Ethernet or WiFi network, if the destination IP address is a broadcast address, then the destination MAC address will also be a broadcast address (FF:FF:FF:FF:FF:FF).

Broadcast addresses are typically used by network nodes to check for the availability of a particular service on the network or if a node does not have an IP address assigned to it. They are also used by servers to advertise their presence to the network nodes.

Broadcasts, while necessary, are disruptive by their very nature. They are used when the sender does not know the exact node it needs to communicate with. Every node that receives the broadcast must process it, even if it is not the intended target. For this reason, the scope of broadcasts should be limited as much as possible. A well-designed network uses VLANs and other network segments to keep broadcast domains relatively small. Routers do not pass broadcasts to other network segments. Switches, however, flood broadcasts out all ports.

Multicast Addresses

A multicast address is an IP address that is used to communicate information from one source to a set of nodes in a network. The set of receivers is predefined as a multicast group. Any nodes that are not part of the multicast group ignore the data transmission. Compared to broadcast addresses, multicast addresses are less disruptive because nodes that are not actively involved in the multicast group simply ignore the packets.

Switches can also be configured to recognize multicast addresses. Some switches and routers run protocols such as Internet Group Management Protocol (IGMP) that keep track of multicast group membership. On these devices, traffic is not sent out any ports that are not associated with the multicast group.

Private IPv4 Addressing

A private IP address is a network address or range of addresses that organizations can use to build internal TCP/IP networks (intranets). Private address ranges are defined by the Request for Comments (RFC) 1918 as:

  • 10.0.0.0 through 10.255.255.255
  • 172.16.0.0 through 172.31.255.255
  • 192.168.0.0 through 192.168.255.255

Most companies divide the private ranges into subnets to provide additional network addresses for their use.

Although they are actual IP addresses, private addresses are not routable on the Internet and are therefore ineligible for use on a public IP network. They have been reserved for private use only.
