Troubleshoot IPv4 and IPv6 Addressing Issues

When IPv4 addresses and subnets are not configured correctly, it can have a great impact on the ability of users to connect to the network resources they need to use.

As with IPv4, misconfigurations in IPv6 addressing can cause problems for user workstations and network devices such as switches and routers.

Common IPv4 Addressing and Subnetting Issues

Some of the most common IPv4 addressing and subnetting issues are:

  • Incorrect IP settings on a host, on the default router, or both.
  • Incorrect DNS server addresses on the host.
  • IP connectivity problem between the DNS server and the host.
  • No match between the subnet masks configured on the host and the router (The host and the router are not connected to the same subnet due to incorrectly set subnet IDs or addresses).
  • Missing IPv4 configuration settings on a host.

Sometimes a networking problem could be because of issues in the routing logic configured on one or more Cisco routers. A routing problem could be caused by the presence of more than one route to forward a packet to the destination, overlap of subnets due to poor subnetting design, overlap routes due to poor addressing, or packets rejected by ACLs.

Other issues can also be mistaken for IP problems. The most common are:

  • Host firewalls blocking traffic, particularly ICMP.
  • Hosts in the wrong VLAN.
  • General Layer 1 and Layer 2 connectivity issues.

Troubleshooting IPv4 Addressing and Subnetting Issues

Consider the following guidelines when you need to troubleshoot IPv4 addressing and subnetting problems:

  • Eliminate non-IP issues such as Physical and Data Link Layer problems, firewalls, and incorrect VLAN configurations.
  • Test network connectivity by using the traceroute and ping commands.
  • Determine the current status at a Cisco router by using the show ip interface and show ip route connected commands.
  • Check the DNS server addresses, subnet masks, and default router settings specified at the host, and correct them if necessary.
  • Ensure that the router and host are connected to the same subnet, and verify that their subnet masks match. Determine the best route for packets and resolve overlap issues caused by overlapping subnets, VLSMs, or poor addressing plans.

Common IPv6 Addressing Issues

Addressing issues can arise in an IPv6 network because of incorrect settings applied at switches, hosts, or routers.

  • The most common issues that occur at a switch include:
    • Missing or incorrect VLAN settings.
    • Incorrect MAC address filtering.
    • Incorrect ACL settings received from the router.
  • The most common issues that occur at a host include:
    • Differences in the prefix length between a host and its default router.
    • The host not being present in the same subnet as its default router.
    • Incorrect default router setting.
    • Incorrect DNS server setting.
  • Most common issues occur at routers because of not adhering to the following IPv6 rules:
    • You must enable IPv6 on every router.
    • All routers must maintain IPv6 routes to all the subnets in the network.
    • The state of router interfaces that are currently in use must be set to "up/up".
      • Note: Each router has two status codes. The first status code indicates the status of the line or connectivity, and the second status code indicates the status of the protocol. The "up/up" state indicates that both the line and protocol status are up, and the device is fully functional.
    • Routers using the same data link must be present in the same IPv6 subnet.

Guidelines for Troubleshooting IPv6 Addressing Issues

Here are some guidelines for troubleshooting IPv6 addressing and configuration issues.

Troubleshooting IPv6 Addressing Issues

Consider the following guidelines when you need to troubleshoot issues involving IPv6 addressing:

  • Tools:
    • The ping and traceroute commands of the Cisco IOS are the most commonly used commands to detect problems in an IPv6 network.
    • Use the show ipv6 route, show ipv6 interface, and show ipv6 neighbors commands to verify IP addressing on an IPv6 network.
  • Some methods to troubleshoot IPv6 addressing issues are:
    • Checking and correcting the static IP settings on a host (such as the default router and DNS server settings).
    • Verifying that the prefix length settings match at the host and its default router (and correcting them when there is a mismatch).
    • Ensuring that the host and router identify the same subnet on a VLAN.
    • Pinging the default router from a host to identify problems in the LAN that incorrectly filter frames or prevent the flow of frames.
    • Checking whether there is a failure in the stateful or stateless autoconfiguration mechanism by ensuring that all hosts have the correct IPv6 address configuration.
    • Using the traceroute command to see if there is a problem in routing packets or if packets are being discarded by ACL filters.
