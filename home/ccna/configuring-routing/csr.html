Configure Static Routing

Although the convenience of having a routing protocol create and update routes is a definite advantage, there will be times when you need the stability of a route that does not change.

Static Routing

Routers forward packets using either route information from the route table entries that you manually configure or the route information that is calculated using dynamic routing algorithms. Static routes refer to the manually configured routing table entries that do not change until they are manually reconfigured. Static routes allow for fixed routing paths in the network.

When there is a change in the network topology, the static route must be updated with the newer route. Static routes are private routes except when they are redistributed by a routing protocol. Unless specifically configured to do so, routers do not update each other with information about static routes. They are configured once, manually, by the administrator. Unlike dynamic routing protocols, static routes use no bandwidth and no CPU cycles for maintenance. Static routes are often used in environments where network traffic is predictable and the network design is simple.

Network and Host Routes

A network route is a route to a specific network ID, whereas a host route is a route to a specific host.

The subnet mask of any route found in a route table implies how many possible hosts exist at the destination. In most cases, the subnet mask of a network route will be the actual subnet mask in use on the destination network. If the route table has additional routes with prefixes that are shorter (have a shorter subnet mask), these are more general and are usually created by aggregating several neighboring networks together. For this reason, the route with the longest (most specific) match is considered to be the more accurate and thus preferred.

The subnet mask for host routes is always composed of all 1s. Thus, the subnet mask for a host route is /32 for IPv4 (and /128 for IPv6) and would be the longest match possible for an address. Using a host route ensures that traffic always follows a specific path to a specific host. Static host routes are often used when redundant paths exist. Configuring a host route for certain hosts ensures that those hosts use the path that you specify.

Loopback interfaces are often created with the /32 subnet mask, making them host routes. In addition, host routes are created for any IP addresses configured on the router.

Floating Static Routes

A floating static route is a static route the router uses to back up a dynamic route. This route must be configured with a higher administrative distance value than the dynamic route it is backing up. It can be used as a replacement when the dynamic route is lost. By default, a router prefers a static route to a dynamic route because the administrative distance of a static route is smaller than the administrative distance of a dynamic route.

For instance, for OSPF the administrative distance is 110. To configure a floating static route, set its administrative distance to 111.

Command Reference: Static Routes

This section contains the commands you will use in the next labs.

Static Routes

The following commands are used in the Static Route labs:

  • clock rate <clock-rate>: Sets the clock rate for a Data Communications Equipment (DCE) interface.
  • configure terminal: Enters global configuration mode from privileged EXEC mode.
  • enable: Enters privileged EXEC mode.
  • end: Ends and exits configuration mode.
  • exit: Exits one level in the menu structure.
  • hostname <host-name>: Sets the device name.
  • interface <type> <number>: Changes from global configuration mode to interface configuration mode.
  • ip address <ip-address> <subnet-mask>: Assigns an IP address to an interface.
  • ipconfig /dg <ip-address>: Used in NetSim to assign a default gateway IP address to a workstation interface.
  • ipconfig /ip <ip-address> <subnetmask>: Used in NetSim to assign an IP address and subnet mask to a workstation interface.
  • ip route <destination-prefix> <destination-prefix-mask> <ip-address> | <interface-type> <ip-address>: Establishes a static route.
  • [no] shutdown: Disables an interface. The no form enables an interface.
  • ping <ip-address>: Sends an Internet Control Message Protocol (ICMP) echo request to the specified address.
  • show ip route: Displays the IP routing table.
  • show running-config: Displays the active configuration file.
