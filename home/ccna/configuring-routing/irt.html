Interpret Routing Tables

Routers depend on their routing tables to make informed decisions about the best path for data transfer between networks. Your ability to read and interpret a routing table can help you determine why certain routers are making certain decisions, and this can be an invaluable troubleshooting tool.

Routing Tables

Each router maintains an IP routing table. This table lists IP address groupings, called IP networks and IP subnets

A hop represents each router through which a packet has to pass as it moves from source to destination address. Next hop represents the IP address of the nearest or the next router in the optimal path.

Routing Protocol Codes

In a routing table entry, the route source helps to identify how the route was learned by the router, which typically has multiple interfaces configured. There are two route source codes for the directly connected interfaces:

  • C: identifies a directly connected network.
  • L: identifies a local route.

The routing table stores information about both directly connected and remote routes. Similar to directly connected networks, the route source identifies how the route was learned for remote networks. For example, the common codes used for remote networks are as follows:

  • S: Stands for the static route. Indicates that the route was manually created by an administrator to reach a specific network.
  • D: Indicates that the route was learned dynamically by the router from another router using the EIGRP routing protocol.
  • O: Indicates that the route was learned dynamically by the router from another router using the OSPF routing protocol.
  • R: Indicates that the route was learned dynamically by the router from another router using the RIP routing protocol.

Administrative Distance

Administrative distance is a feature that routers use to select the best path when there are two or more different routes to the same destination from two different routing protocols. It defines the reliability of a routing protocol. If two or more routing protocols present the router with a route, the route with the lowest administrative distance is placed in the routing table.

Each routing protocol is prioritized in order of most to least reliable (believable) with the help of an administrative distance value. This value is the first criterion that a router uses to determine which routing protocol to use if two protocols provide route information for the same destination. An administrative distance with a lower value gets higher priority or trust. It is a measure of the trustworthiness of the source of the routing information. Administrative distance has only local significance and is not advertised in routing updates. This table lists some common administrative distances.

Routing Protocol Administrative Distance
Directly connected interface 0
Static route (administrator defined) 1
External Border Gateway Protocol (BGP) 20
Internal EIGRP 90
OSPF 110
RIP 120
  • Note: Although static routes have a default administrative distance of 1, the administrator can change this to any value.

Metrics

If two or more routes have the same administrative distance, the router will try to use metrics to decide on the best route. Metrics are values that are associated with specific routes and are ranked from most preferred to least preferred. The parameters used to determine the metrics vary based on the routing protocols used. They can include hop count, cumulative link cost, bandwidth plus delay, and other criteria. The optimal path is the path with the lowest metric, and the selected optimal path is included in the routing table. When multiple paths to the same destination have equal metrics, load balancing is performed on the paths to ensure that no single path is overwhelmed with traffic. This concept is known as equal cost load balancing. There is also unequal cost load balancing. It is supported only by EIGRP and must be configured as it is not enabled by default on Cisco routers.

RIP uses a single routing metric or hop count to measure the distance between the source network and a destination network. A hop count value, which is typically 1, is assigned to each hop in a path from source to destination. When a router receives a routing update that contains a new or changed destination network entry, it adds 1 to the metric value specified in the update and then updates the network address in the routing table. The IP address of the sender is used as the next hop.

Command Reference: Router Basics I and II

Router Basics

The following commands are used in the Router Basics I and II labs:

  • bannder login [<delimiting-character> <message> <delimiting-character>]: Configures a message that is displayed at user login attempts.
  • bannder motd [<delimiting-character> <message> <delimiting-character>]: Configures a message-of-the-day (MOTD) banner that can be used to display a message at user login attempts.
  • configure terminal: Enters global configuration mode from privileged EXEC mode.
  • disable: Returns to user EXEC mode.
  • enable: Enters privileged EXEC mode.
  • enable password <password>: Sets the enable password.
  • enable secret <password>: Sets the enable secret password.
  • end: Ends and exits configuration mode.
  • exit: Exits one level in the menu structure.
  • hostname <host-name>: Sets the device name.
  • line console 0: Accesses console line configuration mode.
  • line vty 0 15: Enters the configuration mode for virtual terminal (Telnet) lines.
  • login: Enables password checking at login.
  • login local: Prompts for a user name and password.
  • logout: Exits fromn the user EXEC mode command-line interface (CLI) and ends the current session.
  • password <password>: Specifies the password that is required for a user to log in.
  • service password-encryption: Applies encryption to all current and future passwords configured on the device.
  • show running-config: Displays the active configuration file.
