Describe basic security concepts

Topic A

Security Concepts

As a CCNA, you should be aware of certain basic security concepts. These concepts form the basis for all of the configuration changes and other actions you will take to establish and maintain a secure network.

Threats

A threat is the possibility of a harmful event such as an intentional attack or exploit, or an accidental event such as a natural disaster. An attack is an action taken against a target with the express intention of causing harm.

Examples of threats include the potential of:

  • Fire, flood, or natural disasters
  • Hackers and terrorists
  • Insufficiently trained users
  • Technical failure of hardware or software

Vulnerabilities

A vulnerability is a weakness that causes a target to be susceptible to attack. For instance, a bank's ATM might be vulnerable to attack if the security camera were to fail.

Exploits

An exploit is a tool or technique attackers use to take advantage of a vulnerability. Examples include:

  • Ransomware, which enables an attacker to extort money from the target.
  • Phishing, where an attacker uses deception to gain information that he or she is not authorized to have access to.
  • SQL injection, which targets database servers with the result of releasing customer information such as credit card numbers or user name/password pairs.
  • Exploit kits, which enable attackers to use automation to identify and act on vulnerabilities.

Type of Attackers

Network attackers have various methods of operation, motivation, and intentions. Attackers can be categorized into different types.

Type of Attacker Description
White Hat Hacker An ethical hacker who discovers and exposes security flaws in the organizational network so they can be resolved.
Black Hat Hacker, also called Cracker A hacker who discovers and exposes security flaws for financial gain or for some other malicious purpose.
Gray Hat Hacker A hacker who acts as a white hat hacker but acts unethically when favorable opportunities arise. For example, a gray hat hacker might be employed as a penetration tester. In the course of his ethical duties, he might find an opportunity for personal gain and act unethically.
Phreaker A hacker who breaks into the telecommunication system without permission. For example, a phreaker might make long distance calls using a dial-up connection without additional charges.
Script Kiddie A novice attacker who has limited skills and uses online utilities to launch attacks, rather than creating custom programs.
Hactivist A hacker with a political intention or with a desire to cause social change. For example, a hacktivist might hack into the website of a political candidate and alter its contents.
Computer Security Hacker A hacker who is knowledgeable about the technical aspects of computer and network security systems. For example, a computer security hacker might attempt to attack a system by fragmenting malicious traffic in such a way that the attack takes place undetected.
Academic Hacker A hacker who exposes security vulnerabilities with the intention of improving the existing infrastructure or establishing a new secure setup. An academic hacker is typically an employee or student at an institution of higher education.
Hobby Hacker A hacker who tends to focus on home computing tasks, such as modifying existing hardware or software - for example, using software without a legitimate license.

Mitigation Techniques

Mitigation Techniques enable IT personnel to address threats, vulnerabilities, and exploits with the goal of protecting organizational resources. Common mitigation techniques include developing and implementing a security program, implementing an identity and access management (IAM) system, and using ACLs to provide fine-grained access control to resources.

Security Programs

One of the most basic mitigation techniques that all organizations can implement is to develop and roll out a comprehensive security program. The goals of a security program address what is commonly known as the CIA triad:

  • Confidentiality: A security program should protect the confidentiality of data.
  • Integrity: A security program must preserve data integrity.
  • Availability: A security program should manage data and make it available to only those who are authorized to access it.

These goals can be accomplished by addressing some common elements, including:

  • User awareness and training
  • Physical access to resources

User Awareness and Training

It is often said that the greatest security vulnerability facing networks is user awareness. People who do not understand how their actions can affect security can cause a great deal of harm by installing unauthorized devices, downloading and installing malicious software, using weak or no passwords, and so forth.

Many organizations counter this vulnerability by providing periodic, role-specific training on security issues. The individual or team responsible for designing a security awareness curriculum should perform or have available a comprehensive list of job roles, use this list to identify the level of training required by each role, and then determine the best course of action for delivering the training awareness information, as well as identify when the training should be administered.

For instance, an organization might decide that a basic user awareness course be delivered to all employees at least once each fiscal year via online or web-based training. Then, those employees who are responsible for protecting customer information might get additional classroom-based training every six months. Finally, IT personnel might be encouraged to pursue technical certifications in the information security (infosec) realm and base promotions, raises, and bonuses on the successful completion of security certification programs.

A cybersecurity-aware organization might also provide awareness information in less formal ways.

  • Posters listing some basic security threats and countermeasures can be placed in strategic locations on the premises.
  • Infosec newsletters or blogs that describe emerging issues and suggested best practices for employees to follow could be distributed three or four times a year.

It is not enough to just provide the training and information. Organizations also need to ensure that retension and compliance meets an acceptable standard. Providing post-training assessment and tracking other metrics such as trends in breaches or other security violations can help ensure that a user awareness and training program is effective, or if it needs to be revised to provide the intended results.

Physical Access Control

Another integral part of an organizational security program is to protect assets from physical threats such as fire, flood, or unauthorized access. Assets include the buildings, computers, media, data, and people that make up the organization. Common physical security controls include:

  • For natural events such as fire, you could install smoke detectors and fire-suppression systems.
  • For human-based threats, you could install CCTV or other video monitoring equipment, hire security guards, and implement a physical access control system that permits access to premises according to job role. Physical separation of the lobby or visitor areas that contain sensitive equipment or information is essential.

AAA

Another popular mitigation technique is to implement identity and access management (IAM) systems, the most common of which are often referred to as AAA (or "triple A") systems. The As stand for:

  • Authentication
  • Authorization
  • Accounting or Auditing

Authentication

Authentication is the process of verifying the identity of the person or device accessing a network device or resource. For Cisco switches and routers, authentication uses a combination of the user ID and password specified by the entity that is attempting to access the device to establish the identity of the devices or users. Cisco devices support both local authentication (using the local lookup database) or remote authentication (using either RADIUS or TACACS+ servers).

Remote authentication uses the challenge and response mechanism. It provides support for messaging and encryption based on the security protocol that you select.

You can configure authentication by defining a named list that consists of authentication methods and applying the list to various interfaces.

A method list must be used to define the types of authentication to be performed and the sequence in which the authentication will be performed. You must apply the method list to a specific interface before any of the defined authentication methods will be performed. There is a default method list that will be automatically applied to all the interfaces when you do not define any other method. However, when you define a method list, it will overrride the default method list.

Authorization

Authorization is the process of assembling a set of attributes that describe the action the user is authorized to perform and then providing suitable access to the user. Authorization provides a method for remote access control and includes one-time authorization or authorization for each service, per-user account list and profile, user group support, and support of IP and Telnet.

Cisco devices provide authorization by using the attributes that are downloaded from AAA servers. Remote security servers, such as RADIUS and TACACS+, allow specific rights to the users by associating attribute-value (AV) pairs. The AV pairs define the rights for the appropriate user. All authorization methods can be defined only through AAA.

  • Note: Do not confuse the concepts of authentication and authorization. A user can be authenticated by a system but have no authorization to perform actions on that system.

Accounting

Accounting provides a method for collecting information, and then logging the information locally or sending the information to a remote AAA server for the purpose of billing, auditing, and reporting. Accounting tracks and maintains a log of every management session used to access network devices such as Cisco switches and routers.

The information collected by acounting can be used to generate reports for troubleshooting and auditing purposes. Accounting enables tracking of the services that the users are accessing as well as the amount of network resources that they are consuming.

When you activate AAA accounting, the network access server will report the user activity in the form of accounting records to the RADIUS or TACACS+ security server, depending on the security method implemented by you. Each accounting record will comprise accounting AV pairs and will be stored on the access control server. All accounting methods can be defined only through AAA.

Access Control

The term access control is pretty self-defining, but it can have many different shades of meaning depending on the context. For Cisco networks, access control refers to a specific means of providing authorization services. The main component of Cisco access control is the ACL.

Access Control Lists (ACLs)

An access control list (ACL) is a security mechanism that helps to control the flow of network traffic. ACLs identify traffic that can be either permitted or denied access to the network. They enable you to define a specific group of traffic and apply a condition based on predetermined criteria.

ACLs are used extensively in Cisco IOS as a way to identify traffic. They are applied by using Cisco IOS statements on a device interface. For instance, an ACL is applied as either an inbound rule or an outbound rule on a router's interface. ACLs validate each packet and determine whether the packet has to be blocked or allowed to pass.

ACLs secure the network by denying unauthorized access. They are commonly used to filter traffic on a router's interface for bandwidth conservation, filter Telnet on virtual terminal lines, segment access to parts of your network, and configure policy-based routing using route maps to control the flow of routing updates.

Two types of ACLs are standard ACLs and extended ACLs.

  • Standard ACLs make filtering decisions based on the source address. For example, you can apply standard ACLs to deny access to a specific host or even an entire network.
  • Extended ACLs check both the source and the destination addresses, plus they check source and destination ports and protocols. For example, if you wanted to deny default access to a File Transfer Protocol (FTP) server, you can apply an extended ACL that cites the destination server as the FTP server and the destination port and protocol as TCP port 21.

Standard and extended ACLs can be numbered or named. Numbered ACLs are identified using numbers in their configuration, and Named ACLs are identified using names. However, both function the same way.

Extended ACLs can also filter traffic based on protocols, port numbers, and other parameters. Both standard and extended ACLs can work with any routing protocol for filtering routes.

Wildcard Masks and ACLs

You can use wildcard masks with ACLs to specify networks, subnets, or groups of hosts that should be permitted or denied access.

  • Use the same rules as you would with OSPF.
  • You can specify just a subset of a particular network - just make sure the devices you want to affect actually fall under the IP range specified by the wildcard mask.
  • You'll get unpredictable results if devices change their IP address (in other words, if they are DHCP clients).

Standard ACLs specify only the source, whereas extended ACLs specify the protocol (IP, TCP, UDP, etc.), source port (optional), destination IP, and destination port (optional).

In ACLs, if you want to match any/all, you could use 0.0.0.0 255.255.255.255, but it is customary to replace the numbers with the word "any". For example: you can write permit 0.0.0.0 255.255.255.255 or permit any.

If you want to match a single host, you could follow it with 0.0.0.0, but it is customary to precede it with the keyword "host". For example, 192.168.1.7 0.0.0.0 can also be written as host 192.168.1.7.

Standard ACL Examples

Here are some examples of using wildcard masks in standard ACLs:

  • To specify that the entire 192.168.7.0/24 subnet is permitted, use this command:
    access-list 1 permit 192.168.7.0 0.0.0.255
  • To permit only host 192.168.7.56, use this command:
    access-list 1 permit 192.168.7.56 0.0.0.0
    Or:
    access-list 1 permit host 192.168.7.56
  • To permit the entire 172.16.4.0/28 subnet (172.16.4.0 - 172.16.4.15), use this command:
    access-list 1 permit 172.16.4.0 0.0.0.15
  • To deny the first 8 addresses of the 192.168.1.0/24 subnet, use this command:
    access-list 1 deny 192.168.1.0 0.0.0.7
  • To permit any IP address, use this command:
    access-list 1 permit 0.0.0.0 255.255.255.255
    Or:
    access-list 1 permit any

Extended ACL Examples

Here are some examples of using wildcard masks in extended ACLs:

  • To specify that the entire 192.168.7.0/24 subnet is permitted to access web server 10.1.1.2, use this command:
    access-list 101 permit tcp 192.168.7.0 0.0.0.255 host 10.1.1.2 eq 80
  • To permit host 192.168.7.56 to use IP to go to any destination, use this command:
    access-list 101 permit ip 192.168.7.56 0.0.0.0 any
    Or:
    access-list 101 permit host 192.168.7.56 any
  • To permit the entire 172.16.4.0/28 subnet (172.16.4.0 - 172.16.4.15) to telnet to 10.0.0.0/8, use this command:
    access-list 101 permit tcp 172.16.4.0 0.0.0.15 10.0.0.0 0.255.255.255 eq telnet
  • To deny the first 8 addresses of the 192.168.1.0/24 subnet from accessing the 172.16.0.0/16 subnet, use this command:
    access-list 101 deny ip 192.168.1.0 0.0.0.7 172.16.0.0 0.0.255.255
  • To permit any source IP address to go to any destination IP address, use this command:
    access-list 101 permit 0.0.0.0 255.255.255.255 0.0.0.0 255.255.255.255
    Or:
    access-list 101 permit any any
