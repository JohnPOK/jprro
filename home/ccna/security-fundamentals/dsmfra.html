Describe Security Measures For Remote Access

Topic E

Remote Access Security

Configuring remote security ensures that your remote-access traffic is safe from unauthorized access.

Remote Access

The term remote access can refer to two conditions:

  • Configuring a network so that users can access its resources from outside the network.
  • Configuring a network device so that IT personnel can manage it from anywhere on the network, or even from outside the network.

Remote Access for Users

Today's workforce includes people who work at the office, people who travel for work, and people who work from home. Wherever they are located, they are likely to need access to network resources.

Remote Access for Management Purposes

You can configure remote access for virtually any Cisco network device.

On routers, Cisco IOS uses the Virtual Terminal (VTY) lines to receive connections related to protocols such as Telnet and SSH. The settings specified on a VTY line apply to session requests that arrive on any of the device interfaces. The typical VTY-level settings include:

  • Timeout Value: This value defines the inactivity timeout for the terminal lines. The parameters of the exec-timeout command are MINUTES and SECONDS, respectively.
  • Password: The combination of the password and login commands, which requires a generic user initiating a Telnet session to the router to specify this password.

To prepare a switch for remote management access, the switch must be configured with an IP address and a subnet mask. The IP address has nothing to do with how switches forward Ethernet frames - it simply exists to support overhead management traffic. Keep in mind that to manage the switch from a remote location on the network (or on another network - basically any time a router sits between you and the switch), the switch must be configured with a default gateway. This is very similar to configuring the IP address information on host devices.

Remote Access Security Considerations

Addressing security issues for remote access depends on the type of remote access you are providing.

  • For user remote access, you must provide access to the appropriate network resources without compromising the network or the data that users send to and receive from remote locations. One of the most popular ways to accomplish this is to set up a virtual private network (VPN).
  • For management remote access, you must protect the network device from unauthorized access. When a switch has been configured with an IP address, it can be accessed via ping and other tools, which can cause a possible vulnerability. A malicious user could discover the IP address and initiate access requests, either gaining remote access or flooding the switch with enough traffic to make it unusable by authorized users. You can implement ACLs to prevent spurious access requests from affecting your network devices.

Tunneling and Encapsulation

The practice of tunneling and encapsulation helps to protect packets as they travel across remote networks.

Encapsulation is the process of enclosing a piece of data in a larger data unit. In fact, encapsulation is the implementation of layered networks, where a unit of Application-layer information is enclosed in a Transport-layer segment, which is in turn enclosed in an Internet-layer packet, and finally enclosed in a Network-Access-layer frame. Each layer adds header information to help move the data to its intended destination.

Tunneling is the process of establishing a private path through one or more public networks by the use of encapsulation. Consider a remote user who needs access to network resources located at the main headquarters. In this set of circumstances, a packet containing private data is enclosed in another packet. For additional protection, the original data is also encrypted. This allows the packet to safely travel across WAN links and the Internet without revealing any of the private data, including the final destination address.

VPNs

A virtual private network (VPN) is a set of security protocols that are implemented on both ends of an insecure network, such as the Internet, to provide secure connectivity to remote sites and partner sites. Common VPN protocols are SSL and IP Security (IPSec).

Internet VPN is a type of VPN used by an enterprise to connect each business to the Internet by using the Internet as a WAN service. An enterprise can connect each business site to the Internet and use VPN to create an Internet VPN. This enables the enterprise's packets to be secured and kept private through encryption and other security mechanisms even when the data is sent over the Internet. An Internet VPN provides important security features to:

  • Prevent anyone (man-in-the-middle) on the Internet from being able to read, copy, or replay the transmitted data and pose as a legitimate user.
  • Verify that the VPN packet originated from a legitimate device and not any device that is used by an attacker.
  • Verify that the packet was intact and not changed during transit on the Internet.

Internet VPN is available worldwide due to the Internet, and the communications are secured by using VPN technology and protocols.

The Dynamic Multipoint VPN (DMVPN) feature on Cisco IOS routers provides a simple and scalable way to create large and small IPSec VPNs by combining Generic Routing Encapsulation (GRE) tunnels, IPsec encryption, and Next Hop Resolution Protocol (NHRP). Spokes in a DMVPN network can register their public IP addresses with the hub during every boot session. Source spokes can query the NHRP database on the hub to obtain the public IP address of the destination spokes. A multipoint GRE tunnel interface will enable a single GRE tunnel to support multiple IPsec tunnels. DMVPN supports two configurations: hub-to-spoke and spoke-to-spoke.

VPN Types

The main types of VPNs are:

  • Client-to-site VPNs, most commonly known as remote access VPNs.
  • Site-to-site VPNs.
  • Host-to-host VPNs.

Client-to-Site (Remote Access) VPNs

A client-to-site VPN is a VPN that connects a single VPN client to a remote site. For instance, an individual who works from home might establish a client-to-site VPN to access the portions of the corporate network that are behind a firewall. This type of VPN is often referred to as a remote-access VPN.

Cisco AnyConnect defaults to a SSL/TLS VPN protocol. You can optionally configure IPSec instead. Here are some general steps for using IPSec to encrypt data on a remote access VPN:

1. Install and configure a VPN client on the remote computer. The Cisco AnyConnect Secure Mobility client or AnyConnect client is software that is placed on the user's PC and creates one end of the VPN remote-access tunnel using SSL. SSL is used by web browsers to provide a secure communication method using port 443. SSL encrypts data that is transferred between the browser and the server after authenticating the user.
  • Note: Cisco also allows Clientless SSL VPN configurations that simply use the browser to establish the VPN connection.
2. On the VPN server, ensure that the Internet Security Association and Key Management Protocol (ISAKMP), also called IKE, is enabled. ISAKMP is the negotiation protocol that allows the IPsec client on the remote PC and the VPN server to agree on how to build an IPsec Security Association. Each ISAKMP negotiation is divided into two sections called Phase 1 and Phase 2.
a. Phase 1 creates a tunnel to protect the authentication and data-tunnel-creation process.
b. Phase 2 creates a tunnel that protects the data that travels across the secure connection.
3. On the remote PC, use the VPN client to authenticate to the VPN server. After authentication, HTTPS messages are allowed to pass through the SSL VPN client. This ensures that all the packets that are sent to the other end of the tunnel are encrypted.

Site-to-Site VPNs

A site-to-site VPN is a VPN tunnel created by two enterprise sites for sending encrypted data between two devices. In a main office/branch office scenario, you might elect to create a site-to-site VPN so that the two networks are connected without the use of a dedicated WAN link.

IPsec is the architecture or framework that needs to be used for security services on the IP networks that connect via a site-to-site VPN. IPsec defines how devices at both the enterprise sites that connect to the Internet can achieve the main goals of VPN -- confidentiality, integrity of data, authentication, and anti-replay.

These are the steps to encrypt data on a site-to-site VPN:

1. The sending VPN device provides the original packet and the session key for encryption and encryption data calculation.
2. The sending device encapsulates the encrypted data into a packet that includes the new IP and VPN headers.
3. The sending device sends the new packet to the destination VPN device.
4. The receiving VPN device uses the encrypted data and session key to run the decryption formula to decrypt the data.

When you are configuring site-to-site VPNs, you should consider the IP subnet design at each location. If two locations use the same subnetting scheme and the same addresses, there will be IP addressing conflicts after the two sites are connected via the VPN. If this happens, you need to reconfigure one of the locations to use a different subnet address.

Host-to-Host VPNs

A host-to-host VPN connects a host at one location to another individual host at another location (or even the same location) via a VPN tunnel. If a remote user establishes a host-to-host VPN, only the resources on the remote host are made available to the remote user. Examples of host-to-host VPNs include:

  • An Internet-facing web server that also needs to connect to an internal SQL server for product and order information. The web server does not need to connect to any other portion of the internal network.
  • An organization that partners with an outsourcing firm for payroll purposes, where all employee data is stored on one server at the organization's site, and all payroll information is stored on one server at the partner site. Neither organization needs to connect to any other portion of their partner's networks.
  • Note: The establishment of host-to-host VPNs is less common in enterprise environments, and is not normally the responsibility of a CCNA.

Summary

In this lesson, you identified common network security concepts and practices. Your ability to protect the data and other assets of your organization will help ensure your success as a CCNA.
