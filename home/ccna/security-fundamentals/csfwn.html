Configure Security For Wireless Networking

Topic D

Configure Wireless Security

Configuring wireless security ensures that your WLAN traffic is safe from unauthorized access.

Wireless Security Considerations

Bounded media provides physical security for the signal. In other words, you have to make a physical connection to the network to exploit it. With unbounded media, all you need is RF, so if you are in range, you can see the signal. If you can see it, you can capture it, and unless it's encrypted, you might be able to read it.

Wireless security controls focus on the available wireless security protocols and how a WLAN is configured.

Wireless Security Protocols

There are several wireless security protocols implemented on today's networks, as described in this table.

Protocol Description
WEP Stands for Wired Equivalent Privacy and is a legacy protocol that should not be used to secure wireless network traffic.
WPA Stands for Wi-Fi Protected Access and was built to replace WEP. It uses a pre-shared key (PSK) of up to 256 bits for authentication. It introduced the Temporal Key Integrity Protocol (TKIP) that changes the key for each transmitted packet and provides message integrity. WPA has been replaced by WPA2 and should not be used to secure wireless traffic.
WPA2 Is an update to and replacement for WPA that offers Advanced Encryption Standard (AES) as an option for encryption. There are two versions: WPA2 personal (which uses PSKs for authentication) and WPA2 enterprise (which uses IEEE802.1x and RADIUS for authentication).
WPA3 Is an update to and replacement for WPA2 and provides enhanced encryption for private and public networks. Like WPA2, it offers personal and enterprise versions. WPA3 personal uses 128-bit encryption and PSKs, while WPA3 enterprise uses 192-bit encryption and 802.1x for authentication.

As you can see, securing wireless traffic includes the implementation of both authentication and encryption. However, there are some instances where you might not want or need to implement authentication, such as in the case of a guest network. In this situation, a captive portal is often used, which requires a user to accept terms of use rather than provide a password. Be aware that most captive portals use unencrypted wireless connections, shifting the security burden onto the client.

WLAN Configuration

To provide security on your WLAN, you need to configure a wireless security protocol. WPA2 is widely deployed, and it is strongly recommended that you implement it or WPA3. You also need to decide if you will use the personal or enterprise version of whichever protocol you use.

On a Cisco WLC, you can use the command line or the GUI tool to configure security protocols on a WLAN. The general steps for configuring WPA2 with a PSK through the GUI are:

1. If necessary, create a WLAN.
2. On the WLAN, enable WPA2 and map it to the interface or interface group that connects to the APs.
3. Configure Layer 2 security to:
a. Enable a WPA2 policy.
b. Select AES or TKIP as the encryption method.
c. Select PSK as the authentication method.
d. Specify the PSK format.
e. Specify the text for the PSK.
4. Save the configuration.
