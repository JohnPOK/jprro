Describe the Components of Wireless LANs

Topic A

Wireless LANs

To configure wireless connectivity, you will need to be familiar with the components that make up a wireless LAN.

Wireless LANs (WLANs)

A wireless LAN enables users to connect to a local network or the Internet, even when they are on the move. WLANs use radio frequency (RF) communication, and they are often implemented to overlay an existing LAN.

Cisco Wireless Architectures

Two main architectures are used for WLANs: autonomous and lightweight

  • An autonomous WLAN architecture places the responsibility for access, security, and overall operation on the wireless access point (WAP), with no separate controller required.
  • A lightweight WLAN architecture places that responsibility on a wireless LAN controller (WLC), and the WAPs have limited firmware, thus functionality.

All WAPs reside in the Access layer of the three-tier model, but when a controller is implemented for a lightweight WLAN, it resides at the Distribution (or possibly Core) layer.

Cisco offers support for both architectures. Some Cisco APs, such as the 2800 series, can be configured as autonomous or as a lightweight AP (LAP).

  • For autonomous WLANs, Cisco Mobility Express and Cisco Aironet® APs are designed for easy deployment while maintaining secure and reliable operation. Designed for businesses that want to deploy wireless connectivity without a lot of IT resources, it also integrates with Cisco DNA Center to provide a form of centralized administration and control.
  • For lightweight WLANs, Cisco's Unified Wireless Network (UWN) architecture offers 802.11 wireless networking solutions for service providers and enterprises. It consists of Cisco WLCs and their associated LAPs, which are controlled by the operating system, using the operating system user interfaces such as Web User Interface, Command-Line Interface (CLI), and Cisco Wireless Control System (Cisco WCS). UWN simplifies deployment and management of large-scale wireless LANs and provides a unique best-in-class security infrastructure.

WLAN Components

Typical WLAN components include clients, APs, WLCs, access and trunk ports (when you assign WLAN components to VLANs), and Link Aggregation Groups (LAGs).

APs

An AP is a device that serves as the interface between wireless and wired networks or as the center point of a standalone wireless network. In large installations, wireless users within the radio range of an AP can roam throughout a facility while maintaining seamless, uninterrupted access to the network. Each AP contains one, two, or three radios.

AP Modes

APs can operate in a variety of modes.

LAP modes includes the following, depending on the model of LAP and the version of Cisco IOS being used.

LAP Mode Description
Local mode This is the default mode of operation, where the LAP maintains a tunnel to its associated WLC and just forwards all traffic through it. Because the WLC is responsible for switching and forwarding the traffic, if a local mode LAP loses its connection to the WLC, it stops forwarding traffic and starts looking for a controller.
Remote Edge (lightweight) Access Point (REAP) mode and its derivatives REAP mode addresses the scalability issues inherent with using Local mode. For instance, it may not be feasible to place a WLC at multiple branch locations, but if you lose the WAN link that connects the LAPs with a remote WLC, then the LAP's entire branch location would stop forwarding traffic. LAPs operating in REAP mode do not always require a connection to the WLC, and they can switch local WLAN traffic without relying on the controller. The functionality of REAP was later expanded to H-REAP (Hybrid-REAP) and then FlexConnect, which is the most recent version.
Bridge mode Bridge mode enables you to bridge the WLAN and the wired infrastructure.
SE-Connect mode Spectrum Expert (SE) Connect mode enables you to connect to LAPs that support the Cisco CleanAir technology by using Cisco Spectrum Expert and gather vital information about the RF spectrum surrounding the LAP. A LAP operating in SE-Connect mode does not broadcast an SSID or service any WLAN clients. This mode is used strictly for troubleshooting purposes.
Sniffer mode Similar to SE-Connect mode, Sniffer mode is strictly for troubleshooting purposes. Sniffer mode passively monitors the WLAN environment by using a specifically configured channel and tunnels all 802.11 WLAN traffic to an endpoint on your network, where you can use protocol analysis tools such as Wireshark to review the packets and diagnose issues.
Rogue Detector mode Again similar to SE-Connect mode & Sniffer mode, this mode does not service WLAN clients. Rogue Detector mode connects to your wired infrastructure, usually over a trunk link, and monitors traffic moving among VLANs. LAPs running in Rogue Detector mode watch for any MAC addresses that other LAPs have marked as wireless rogue devices. So, LAPs oprating in Local mode or a REAP-based mode passively watch and report rogue APs and rogue WLAN clients, while LAPs in Rogue Detector mode watch the wired network for the MAC addresses of those same rogue APs and rogue WLAN clients.
Monitor mode Monitor mode is a feature designed to enable LAPs to exclude themselves from handling data traffic between clients and the infrastructure. Instead, they act as dedicated sensors for location-based services (LBS), rogue access point detection, and intrusion detection (IDS). When APs are in Monitor mode, they do not serve clients, and they continuously cycle through all configured channels listening to each channel for approximately 60 ms.
Root mode Root mode is the default mode, and indicates an AP that is directly connected to the wired LAN backbone.
Root Bridge mode When you configure an AP to act as a bridge, you can select either Root Bridge or Non-Root Bridge mode. Similar to switches using STP, the Root Bridge mode AP connects Non-Root Bridge APs to the wired LAN. It can also have client devices connected to it.
Non-Root Bridge mode Similar to switches using STP, the Non-Root Bridge mode AP is connected to a Root Bridge mode AP to access the wired LAN, and it can have client devices connected to it.
Workgroup Bridge mode When an AP is configured in Workgroup Bridge (WGB) mode, it can provide wireless connectivity to LAPs for wired clients. This mode is optimal for connecting fewer than 20 wired clients to a LAP.
Scanner mode When an AP is in Scanner mode, it does not support clients but instead acts as a network monitoring device.
Spectrum mode Spectrum mode is the autonomous AP equivalent of SE-Connect mode.
Repeater mode An AP configured in Repeater mode provides a wireless uplink to the wired network by connecting as a client to a Root mode AP. Clients see the Repeater mode AP as a regular AP. This mode reduces wireless throughput and might not work as expected with non-Cisco client devices. Therefore, it is recommended for use only when necessary to extend the range of the wireless infrastructure.

Other Wireless Components

Additional wireless LAN components are described in this table.

WLAN Component Description
WLCs WLCs manage LAPs, and they can be hardware devices or software-based controllers.
Access and trunk ports Access and trunk ports on APs operate the same as they do in wired VLANs.
LAGs On Cisco WLCs, you can configure link aggregation groups (LAGs) to combine the controller ports into one channel to simplify configuration, increase bandwidth between the WLAN and wired LAN, and provide load balancing and redundancy. Only one LAG is supported on each WLC.

Wi-Fi Channels

Wi-Fi channels are the common name used for WLAN channels. APs have one to three radios incorporated in them. The IEEE 802.11 standard provides several distinct RF ranges for Wi-Fi communications, including 900 MHz, 2.4 GHz, 3.6 GHz, 4.9 GHz, 5 GHz, 5.9 GHz, and 60 GHz.

Each of these ranges is further divided into channels. The standards refer to numbered channels spaced evenly apart, with the channel number being the midpoint of the channel's frequency range. You can however, bond channels for higher throughput. In this case, the channel number is the center frequency of the bonded group.

To prevent the interference caused by channel overlap, the Wi-Fi protocol calls for channel seperation of 16.25 to 22 MHz, which corresponds to three non-overlapping channels on 2.4 GHz: 1, 6, and 11.

There are circumstances where you might use subsets of frequencies to provide additional channels.

Wi-Fi Network Standards

Numerous Wi-Fi standards have been developed. This table briefly describes them.

Wi-Fi Standard Description
802.11a This standard uses the 5.2 to 5.8 GHz range. Channel width is 20 MHz. Its theoretical maximum rate is 54 Mbps. A distinguishing feature is that this standard offered eight non-overlapping channels in the U.S., making it relatively easy to avoid interference from other wireless installations.
802.11b This standard uses only the 2.4 GHz band. Channel width is 22 MHz. It has a theoretical maximum rate of 11 Mbps.
802.11g This standard uses only the 2.4 GHz band. Channel width is 20 MHz. Its theoretical maximum rate is 54 Mbps. It is backward compatible with 802.11b, but 802.11b devices connecting to an 802.11g AP will operate at 802.11b speeds.
802.11n Also referred to as Wireless-N, it uses either the 2.4 GHz band or the 5 GHz band, with the majority of devices at 2.4 GHz. Channel width is 20 or 40 MHz. It is fully compatible with 802.11b and 802.11g. Speeds of up to 300 Mbps are possible.
802.11ac The latest of the standards to be released, it uses only the 5 GHz band, lowering the risk of interference. Channel width is 20, 40, or 80 MHz, twice the width of 802.11n. It is fully backward compatible with 802.11n, and in environments where all devices support it, can provide at least 1 Gbps for multiple-station throughput and at least 500 Mbps for a single link.

Service Set Identifiers (SSIDs)

Service Set Identifiers (SSIDs) are simply names for Wi-Fi networks. For instance, if you go to a coffee shop or other business that offers wireless to their patrons, your mobile phone will probably prompt you to connect to a Wi-Fi network, and then lists the SSIDs for any local WLANs it detects. The reason that devices can detect WLAN SSIDs is because the APs broadcast them. If there is no SSID broadcast, the networks will not be discoverable.

Depending on how the AP is configured, you might need a Wi-Fi password to gain access to the WLAN.

Encryption

Wireless networks, like all networks, require security to protect the components and the data stored on them. For WLANs, one of the primary security controls is encryption, or the process of encoding information so that only those authorized to view it can view it. Even if a transmission is intercepted by an unauthorized party, the contents of that transmission cannot be read.

Wi-Fi security is often implemented by using one of the following security standards:

  • Wired Equivalent Privacy (WEP), which is a weak implementation of the RC4 encryption algorithm. WEP keys were 24 bits shorter than indicated and used a 24-bit Initialization Vector (I.V.). It has been replaced by WPA.
  • Wi-Fi Protected Access (WPA), which was designed specifically to address the shortcomings of WEP. It uses the TKIP protocol to change the encryption key for every packet. It also adds message integrity and a 48-bit I.V., and it uses true key sizes.
  • WPA2, which might require firmware changes to run successfully, also uses a 48-bit key and true key sizes. However, it uses a variant of the AES encryption algorithm called CCMP instead of RC4. There are two flavors of WPA2:
    • WPA2 Personal uses a shared password for authentication.
    • WPA2 Enterprise uses a user name/password pair or digital certificates for authentication.
