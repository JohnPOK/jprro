Configure Clients for WLAN Access

Now that you know a bit more about WLANs, their components, and how you connect to them, you can begin to set up WLAN access for client devices.

As with switches and routers, you need to connect to WLAN components so that you can configure and manage them. Some of the connection mechanisms are similar to those for wired networking devices, but there are a few additional options you should be aware of.

WLAN Creation

Creating a WLAN involves configuring autonomous APs or LAPs and their associated WLCs. The process for creating a WLAN contains several stages:

1. Configure the WLC for basic operation.
2. Configure LAPs.
3. Register LAPs on the WLC.

You can use either the console port or the service port to configure a WLC.

  • If you use the console port, you will use the CLI to work through the configuration wizard.
  • If you use the service port, you will use the web-based GUI Configuration Wizard.

The Configuration Wizard

The Configuration Wizard prompts you to enter the following information:

  • On the System Information page:
    • System name
    • Administrative user name
    • Administrative user password
  • On the SNMP Summary page, which of the SNMP modes you want to enable, if any:
    • SNMP v1 mode
    • SNMP v2c mode
    • SNMP v3 mode
  • On the Service Interface Configuration page:
    • Whether or not you want the controller's service port to obtain an IP address automatically.
    • If you want to manually configure the IP address for the service-port interface, the IP address, and the netmask.
  • On the LAG Configuration page, whether or not you want to enable LAG.
  • On the Management Interface Configuration page:
    • The VLAN ID associated with the management interface. This should match the switch interface configuration.
    • The IP address, netmask, and default gateway for the management interface.
    • The port number and backup port for the management interface.
    • If automatic IP addressing is implemented on the LAN, the addresses of the primary and secondary DHCP servers.
  • On the AP-Manager Interface Configuration page, the IP address of that interface.
  • On the Miscellaneous Configuration page:
    • The name of the mobility group or RF group you want to place the controller in.
    • What country codes apply to the WLC and associated controllers. You can assign multiple country codes on this page.
  • On the Virtual Interface Configuration page:
    • The IP address you want to use for the virtual interface. This should be an unassigned address.
    • The name of the DNS server that should be used to verify web authorization certificates.
  • On the WLAN Configuration page:
    • The WLAN profile name.
    • The WLAN SSID.
  • On the RADIUS Server Configuration page, information related to the RADIUS authentication server.
  • On the 802.11 Configuration page, which types of LAPs you want to support and whether or not you want to use the auto-RF feature.
  • On the Set Time page, date and time information.

Security Settings

WLAN security settings include:

  • Authentication
  • Encryption
  • Implementing wireless VLANs and assigning multiple SSIDs
  • Wireless IDS configuration

Quality of Service (QoS)

Quality of Service (QoS) is the ability of a network to provide a specific level of service to a group of users or applications, even if providing that level of service degrades performance and availability for other users or applications. QoS:

  • Supplies dedicated bandwidth to critical users and applications.
  • Controls parameters that affect real-time applications, such as jitter and latency.
  • Manages bandwidth to minimize network congestion.
  • Provides a consistent flow of network traffic.
  • Sets and enforces traffic priorities.

QoS Profiles

For Cisco WLANs, Qos is based on the Wi-Fi MultiMedia (WMM) and Cisco Compatible Extensions (CCX) programs. Cisco UWM products use WMM, and the controllers support four QoS profiles:

  • Platinum/Voice: Ensures the highest QoS for voice over wireless.
  • Gold/Voice: Supports high-quality video applications.
  • Silver/Best Effort: As the default setting, provides normal bandwidth for clients.
  • Bronze/Background: Provides the lowest bandwidth for guest services.

Advanced WLAN Settings

The Advanced menu of the controller GUI provides access to the following administrative tasks:

  • Working with AP groups
  • Load balancing between APs
  • Band selection thresholds (redirecting clients away from 2.4 GHz to less-congested 5 GHz)
  • Provides a consistent flow of network traffic.
  • Radio configuration

Summary

In this lesson, you configured wireless connectivity. By setting up APs or WLCs and LAPs, you can ensure that users can access network resources no matter where they might be in your organization's physical structure.
